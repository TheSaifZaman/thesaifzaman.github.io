Chapter 4

Implementing a GPT Model from Scratch

8 Lessons 8-Question Quiz ~45 min read Core Architecture

With attention mechanisms fully understood from Chapter 3, we now have everything needed to assemble a complete GPT model. This chapter builds GPT-2 Small from first principles — every component from scratch. We will implement layer normalization, the GELU activation, the feed-forward network, residual connections, the full transformer block, and finally the end-to-end model with text generation. By the end you will have a working 124M-parameter GPT-2 in under 100 lines of PyTorch.

4.1

GPT Architecture Overview

A GPT model is a stack of transformer blocks preceded by an embedding layer and followed by a linear projection that maps back to vocabulary size. The full forward pass flows through a well-defined sequence of operations: token IDs enter the model, get embedded into dense vectors, positional information is added, the sequence passes through N identical transformer blocks, a final LayerNorm is applied, and a linear layer (the "language model head") projects to logits over the vocabulary. Softmax over those logits gives a probability distribution for the next token.

GPT-2 Small uses the following configuration: vocab_size=50257, context_length=1024, d_model=768, num_heads=12, num_layers=12, dropout=0.1. These values are not arbitrary — the vocabulary comes from BPE tokenisation on WebText, d_model=768 is a balance between expressivity and computational cost, and 12 layers is empirically sufficient for strong language modelling at this scale.

GPT-2 Small has approximately 124 million parameters total. Roughly 38 million live in the embedding layers (token + positional), and the remaining 85 million are distributed across the 12 transformer blocks (attention heads, feed-forward networks, layer norms). A key design choice — weight tying — shares the token embedding matrix with the output projection, saving ~38M parameters that would otherwise be needed for a separate lm_head weight matrix.

Understanding the architecture as a whole before diving into components is essential. Every component serves a specific purpose: embeddings convert discrete tokens to continuous space, transformer blocks build contextual representations through attention and position-wise FFN, and the lm_head maps the final representation back to a probability distribution over the vocabulary. The design is clean, composable, and fully parallelizable — the defining reason transformers overtook RNNs.

4.2

Layer Normalization

Layer Normalization (LayerNorm) normalizes activations across the feature dimension of each token independently. For a token's activation vector x of dimension d, it computes the mean and standard deviation across those d values, subtracts the mean, divides by the std, then applies learnable scale (γ) and shift (β) parameters. Formally: y = (x − mean) / (std + ε) · γ + β, where ε is a small constant (1e-5) for numerical stability. Both γ and β are learned during training — they allow the model to undo the normalization if that is what minimizes the loss.

GPT-2 uses the Pre-Norm variant of LayerNorm: it is applied before each sublayer (attention and FFN), inside the residual branch. This is in contrast to the Post-Norm in the original "Attention Is All You Need" paper, where LN was applied after the residual addition. Pre-Norm training is empirically more stable for deep networks because the residual stream maintains its scale throughout the forward pass — gradients do not explode or vanish.

Why not BatchNorm? BatchNorm normalizes across the batch dimension — it needs statistics computed over many samples simultaneously. For variable-length sequences (common in NLP), sequences must be padded to equal length, which contaminates the batch statistics with padding values. Furthermore, small batch sizes lead to noisy batch statistics. LayerNorm computes statistics per-token, per-example — it is completely independent of other samples in the batch and works equally well at any batch size, including batch size 1 during inference.

class LayerNorm(nn.Module):
    def __init__(self, d_model, eps=1e-5):
        super().__init__()
        self.gamma = nn.Parameter(torch.ones(d_model))   # scale (learned)
        self.beta  = nn.Parameter(torch.zeros(d_model))  # shift (learned)
        self.eps   = eps

    def forward(self, x):
        mean = x.mean(-1, keepdim=True)
        std  = x.std(-1, keepdim=True, unbiased=False)
        return self.gamma * (x - mean) / (std + self.eps) + self.beta
LayerNorm — Activation Distribution Before & After
Each bar represents one of 8 activation values for a single token. Before LayerNorm, activations are scattered with arbitrary mean and variance. After LayerNorm, they are centered (mean ≈ 0) and scaled (std ≈ 1) — giving the model a consistent input scale regardless of what the previous layer output.
4.3

GELU Activation Function

GELU (Gaussian Error Linear Unit) is the activation function used in GPT-2's feed-forward networks. Unlike ReLU which hard-zeros all negative inputs, GELU multiplies each input by the probability that a standard normal random variable is less than or equal to that value: GELU(x) = x · Φ(x), where Φ is the Gaussian CDF. This gives GELU a smooth, non-zero gradient for negative inputs — it attenuates negative values rather than eliminating them.

In practice, computing the exact CDF is expensive, so PyTorch uses a fast polynomial approximation: GELU(x) ≈ 0.5 · x · (1 + tanh(√(2/π) · (x + 0.044715 · x³))). This approximation is extremely close to the true GELU and widely used. At x=0, GELU(0)=0. For large positive x, GELU(x)≈x (like ReLU). For small negative x, GELU(x) slightly attenuates the value, and for large negative x it approaches 0 — but smoothly.

Why does GELU outperform ReLU in transformers? ReLU kills all negative activations completely (dead neuron problem: if a neuron always outputs negative pre-activations, it never receives a gradient and never learns). GELU's smooth curve ensures gradients remain non-zero everywhere. Additionally, GELU's stochastic interpretation (acting as a random dropout that depends on the input value) provides implicit regularization. Empirically, GELU consistently outperforms ReLU on large transformer benchmarks, and has become the standard activation for language models.

4.4

Feed-Forward Network (FFN)

Every transformer block contains a two-layer feed-forward network (FFN) applied to each token independently after the attention sublayer. The structure is: Linear(d_model → 4×d_model) → GELU → Linear(4×d_model → d_model). In GPT-2 Small this means 768 → 3072 → 768. The intermediate expansion ratio of 4× comes from the original "Attention Is All You Need" paper and has been kept in virtually every subsequent transformer architecture. It gives the model a large "scratchpad" dimension to compute complex nonlinear transformations of the token representation.

A critical point: the FFN processes each token independently and identically — there is no communication between token positions inside the FFN. Inter-token communication is entirely handled by the attention sublayer. This clean separation of concerns is elegant: attention gathers information from across the sequence, while the FFN processes and transforms that information for each token individually. The FFN can be thought of as storing "factual knowledge" — large models encode enormous amounts of world knowledge in their FFN weights.

Parameter count for a single FFN: two linear layers contribute 768×3072 + 3072×768 = ~4.7M parameters. Across 12 blocks: ~56.6M parameters. This is the single largest component of GPT-2 Small — roughly 67% of all transformer block parameters live in the FFN layers. Adding dropout after the second linear layer provides regularization, preventing the model from memorizing training data.

class FeedForward(nn.Module):
    def __init__(self, d_model):
        super().__init__()
        self.net = nn.Sequential(
            nn.Linear(d_model, 4 * d_model),  # 768 → 3072
            GELU(),
            nn.Linear(4 * d_model, d_model),  # 3072 → 768
            nn.Dropout(0.1)
        )

    def forward(self, x):
        return self.net(x)  # each token processed independently
FFN vs Attention — Division of Labor: Attention = "which tokens should communicate, and how?" FFN = "given what I have gathered, what nonlinear transformation should I apply to my own representation?" The FFN runs after attention in each block, applying the same learned transformation to every position independently.

4.5

Residual (Skip) Connections

Residual connections, also called skip connections, add the input of a sublayer directly to its output: output = x + sublayer(LayerNorm(x)). The identity shortcut x bypasses the sublayer entirely. This sounds deceptively simple, but it is one of the most important architectural innovations in deep learning. Introduced in ResNet for image classification in 2015 and adopted universally since, residual connections solve the vanishing gradient problem that previously made training networks deeper than ~20 layers extremely difficult.

Why do residuals help gradients? In a standard deep network, the gradient of the loss with respect to an early layer involves a product of many Jacobian matrices — one per layer. If those Jacobians have spectral norms less than 1 (which they often do), the product shrinks exponentially toward zero. With residuals, the gradient through the identity path is always 1: ∂output/∂x = 1 + ∂sublayer/∂x. Even if the sublayer gradient is near zero, the gradient still flows back with magnitude 1. This makes training a 12-layer GPT-2 as stable as training a 2-layer network — the depth is effectively "free".

In GPT-2, there are two residual connections per transformer block: one around the attention sublayer and one around the FFN sublayer. Both use the Pre-Norm pattern: x = x + drop(attn(ln(x))) and x = x + drop(ffn(ln(x))). The dropout inside each residual branch provides additional regularization. Because the residual stream (the x variable) flows straight through the entire 12-block stack, the model's output is essentially the sum of many small corrections applied by each block — rather than a complex chain of transformations where early information is irreversibly transformed.

Gradient Flow: With vs Without Residuals
Without residuals (left), gradient magnitude shrinks exponentially through 12 layers — the first few layers receive almost no gradient signal and learn very slowly. With residuals (right), the gradient highway maintains full magnitude at every layer depth, enabling all 12 layers to learn effectively simultaneously.
4.6

The Full Transformer Block

A transformer block combines all the components seen so far: two LayerNorms, a multi-head attention sublayer, a feed-forward sublayer, two residual connections, and dropout. The operations in order: (1) LayerNorm the input, (2) pass through multi-head attention, (3) apply dropout, (4) add the original input (residual), (5) LayerNorm the result, (6) pass through FFN, (7) apply dropout, (8) add input of step 5 (residual). The output has the same shape as the input — [B, T, d_model] — making blocks stackable identically.

GPT-2 uses the Pre-Norm ordering (LayerNorm before the sublayer), which differs from the original 2017 Transformer paper that used Post-Norm (LayerNorm after residual addition). The Pre-Norm ordering makes training more numerically stable. In Post-Norm, the residual stream can grow unboundedly, leading to training instability in very deep networks. Pre-Norm keeps the residual stream's scale stable throughout training because the LayerNorm normalizes activations before they enter each sublayer, not after the potentially large residual addition.

class TransformerBlock(nn.Module):
    def __init__(self, cfg):
        super().__init__()
        self.ln1  = LayerNorm(cfg['d_model'])
        self.attn = MultiHeadAttention(cfg['d_model'], cfg['num_heads'])
        self.ln2  = LayerNorm(cfg['d_model'])
        self.ffn  = FeedForward(cfg['d_model'])
        self.drop = nn.Dropout(cfg['dropout'])

    def forward(self, x):
        x = x + self.drop(self.attn(self.ln1(x)))  # residual + attention
        x = x + self.drop(self.ffn(self.ln2(x)))   # residual + FFN
        return x

4.7

The Full GPT Model

With all components in place, assembling the full GPT model is straightforward. The model holds an embedding layer for tokens, a separate embedding layer for positions (up to context_length), dropout, a stack of N TransformerBlocks (as an nn.Sequential), a final LayerNorm, and the linear language model head (lm_head). The forward pass takes integer token IDs of shape [B, T], generates position indices [0, 1, ..., T-1], embeds both, sums them, and passes through the entire stack before projecting to logits.

A subtle but important implementation detail: weight tying. The token embedding matrix (tok_emb.weight) and the lm_head weight matrix represent the same conceptual object — a mapping between token IDs and d_model-dimensional vectors. The embedding maps IDs to vectors (rows of a 50257×768 matrix), and lm_head maps vectors back to IDs (matrix product). Sharing the same weight matrix saves ~38.6M parameters (50257 × 768) and, importantly, improves performance: the model learns a single consistent token representation space rather than two separate potentially-conflicting ones.

The positional embedding uses a learned table of 1024 positions (one embedding per position), rather than the fixed sinusoidal embeddings from the original Transformer paper. GPT-2's learned positional embeddings allow the model to learn the most useful position representations from data. The token and position embeddings are simply added together (not concatenated), which preserves the d_model dimension and allows attention to jointly use content and position information.

class GPTModel(nn.Module):
    def __init__(self, cfg):
        super().__init__()
        self.tok_emb = nn.Embedding(cfg['vocab_size'],     cfg['d_model'])
        self.pos_emb = nn.Embedding(cfg['context_length'], cfg['d_model'])
        self.drop    = nn.Dropout(cfg['dropout'])
        self.blocks  = nn.Sequential(*[TransformerBlock(cfg)
                                        for _ in range(cfg['num_layers'])])
        self.ln_f    = LayerNorm(cfg['d_model'])
        self.lm_head = nn.Linear(cfg['d_model'], cfg['vocab_size'], bias=False)

        # Weight tying: share embedding and output projection
        self.lm_head.weight = self.tok_emb.weight

    def forward(self, idx):
        B, T = idx.shape
        tok = self.tok_emb(idx)                       # [B, T, d_model]
        pos = self.pos_emb(torch.arange(T))           # [T, d_model]
        x   = self.drop(tok + pos)
        x   = self.blocks(x)
        x   = self.ln_f(x)
        return self.lm_head(x)                         # [B, T, vocab_size]
Weight Tying Detail: Setting self.lm_head.weight = self.tok_emb.weight makes them share the same underlying tensor. When PyTorch counts parameters, it counts this matrix once (~38.6M). Without weight tying, GPT-2 Small would need ~162M parameters instead of 124M — a 31% increase for no empirical gain.
4.8

Text Generation (Greedy Decoding)

GPT generates text autoregressively: the model takes a sequence of tokens as input, produces a probability distribution over the vocabulary for the next token, samples or picks that token, appends it to the sequence, and repeats. At each step, the full context window (all previously generated tokens plus the prompt) is fed into the model — GPT is not recurrent, so there is no hidden state to carry forward; the entire sequence is re-processed each step. This is computationally expensive but enables the model to attend to all previous context at every step.

Greedy decoding always picks the highest-probability next token: argmax(softmax(logits[-1])). This is the simplest strategy and is deterministic — given the same prompt, you always get the same output. Greedy decoding is fast but can produce repetitive or suboptimal text, since maximizing each individual step does not guarantee the globally optimal sequence. In practice, temperature sampling (dividing logits by a temperature T before softmax) and top-k / top-p (nucleus) sampling produce more natural and diverse text.

One important detail: if the generated sequence grows longer than context_length (1024 for GPT-2 Small), we must crop it before feeding into the model. The positional embedding table only has entries for positions 0 to 1023 — passing a longer sequence would require position IDs that do not exist in the embedding table. In practice, sequences rarely reach this limit during generation, but production systems handle this by either truncating or using techniques like sliding window attention.

def generate(model, idx, max_new_tokens, cfg):
    for _ in range(max_new_tokens):
        # Crop to context window if needed
        idx_crop = idx[:, -cfg['context_length']:]
        with torch.no_grad():
            logits = model(idx_crop)          # [B, T, vocab_size]
        logits    = logits[:, -1, :]          # last token: [B, vocab_size]
        probs     = F.softmax(logits, dim=-1)
        idx_next  = torch.argmax(probs, dim=-1, keepdim=True)  # greedy
        idx       = torch.cat([idx, idx_next], dim=1)
    return idx

Chapter 4 — Key Takeaways

  • GPT architecture = Token Embedding + Positional Embedding + N × Transformer Blocks + Final LayerNorm + lm_head. GPT-2 Small uses N=12 blocks, d_model=768, vocab_size=50257, yielding ~124M parameters.
  • LayerNorm normalizes activations across the feature dimension (not batch) per token, using learned γ (scale) and β (shift). GPT-2 uses Pre-Norm: LN is applied before each sublayer for more stable deep network training.
  • GELU activation outperforms ReLU in transformers because it is smooth everywhere and allows non-zero gradients for negative values — no dead neuron problem. Formula: GELU(x) = 0.5 · x · (1 + tanh(√(2/π) · (x + 0.044715x³))).
  • Feed-Forward Network expands d_model by 4× (768 → 3072) then contracts back — the largest component of GPT-2 at ~56.6M parameters across 12 blocks. It processes each token independently after attention.
  • Residual connections in the form output = x + sublayer(LN(x)) prevent vanishing gradients in deep stacks by giving gradients a direct identity path (gradient = 1) at every block depth.
  • Weight tying shares the token embedding matrix (tok_emb.weight) with the output projection (lm_head.weight), saving ~38.6M parameters and enforcing a consistent token representation space.
  • Greedy text generation is autoregressive: encode prompt → forward pass → argmax last-token logits → append → repeat. The context window is cropped if the sequence exceeds context_length=1024.
