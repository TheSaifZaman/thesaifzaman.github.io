What We're Building

We will train a character-level GPT on Tiny Shakespeare — 1 MB of public-domain text. The model is intentionally tiny so you can train it in minutes on a laptop CPU. Every concept maps directly to a chapter you've already studied.

Goal: By the end of this capstone, you'll have run python train.py, watched the loss fall in your terminal, then run python generate.py and seen your model produce Shakespeare-ish text — all from scratch, in ~300 lines of Python you wrote yourself.

config.py — Hyperparameters

Centralise every hyperparameter in one config dict. This makes experiments easy — you can sweep values without hunting through multiple files.

Why a config dict? In real codebases (and in the book's code), all hyperparameters live in one place. When you change d_model here, every module that imports GPT_CONFIG automatically picks up the new value.
# config.py — Mini GPT Hyperparameters GPT_CONFIG = { 'vocab_size': 65, # character-level: 65 unique chars in tiny shakespeare 'context_length': 64, # tokens of context 'd_model': 128, # embedding + hidden dimension 'num_heads': 4, # attention heads 'num_layers': 4, # transformer blocks 'dropout': 0.1, 'bias': False, } TRAIN_CONFIG = { 'batch_size': 32, 'max_iters': 5000, 'eval_interval': 500, 'lr': 3e-4, 'weight_decay': 0.1, 'grad_clip': 1.0, 'device': 'cuda' if __import__('torch').cuda.is_available() else 'cpu', 'checkpoint_path': 'checkpoint.pt', }

data.py — Tokenization & Dataset

Download Tiny Shakespeare, build a character-level vocabulary, encode the text to integers, split into train/val sets, and wrap in a PyTorch Dataset. This is the pipeline from Chapter 2.

Character-level vs BPE: GPT-2 uses BPE tokenization (50,257 tokens). For this project we use the simpler character-level approach: each unique character is one token. With only 65 characters, our vocabulary fits in a tiny embedding table. See the "What's Next" section in Tab 5 to upgrade to tiktoken BPE.
# data.py — Data loading and tokenization import torch import urllib.request def download_data( url='https://raw.githubusercontent.com/karpathy/char-rnn/master/data/tinyshakespeare/input.txt', path='input.txt' ): """Download tiny Shakespeare dataset.""" try: urllib.request.urlretrieve(url, path) print(f"Downloaded to {path}") except Exception as e: print(f"Download failed: {e}. Place a text file at {path} manually.") def build_vocab(text): """Build character-level vocabulary.""" chars = sorted(set(text)) stoi = {ch: i for i, ch in enumerate(chars)} itos = {i: ch for i, ch in enumerate(chars)} return stoi, itos, len(chars) def encode(text, stoi): return [stoi[c] for c in text] def decode(ids, itos): return ''.join(itos[i] for i in ids) class CharDataset(torch.utils.data.Dataset): def __init__(self, data, context_length): self.data = data self.ctx = context_length def __len__(self): return len(self.data) - self.ctx def __getitem__(self, idx): chunk = self.data[idx : idx + self.ctx + 1] return chunk[:-1], chunk[1:] # input, target (shifted by 1) def get_loaders(config, train_config): import os if not os.path.exists('input.txt'): download_data() with open('input.txt', 'r') as f: text = f.read() stoi, itos, vocab_size = build_vocab(text) print(f"Vocab size: {vocab_size}, Text length: {len(text):,} chars") config['vocab_size'] = vocab_size # update config with actual vocab size data = torch.tensor(encode(text, stoi), dtype=torch.long) n = int(0.9 * len(data)) train_data = data[:n] val_data = data[n:] train_ds = CharDataset(train_data, config['context_length']) val_ds = CharDataset(val_data, config['context_length']) loader_kwargs = dict(batch_size=train_config['batch_size'], shuffle=True, drop_last=True) return ( torch.utils.data.DataLoader(train_ds, **loader_kwargs), torch.utils.data.DataLoader(val_ds, **loader_kwargs), stoi, itos )
The shifted-by-1 trick (Ch 2 & 5): __getitem__ returns (chunk[:-1], chunk[1:]). The input is tokens 0…63 and the target is tokens 1…64. At each position the model predicts the next character. This turns a single sequence into 64 training examples — that's why cross-entropy loss is computed over every position simultaneously.

model.py — GPT Architecture

The complete model in one file — five classes, zero external dependencies beyond PyTorch. Each class maps to a chapter you've read.

Class map — what each class does
LayerNorm
Chapter 4 — Implementing GPT from Scratch

Manual Layer Normalisation — normalises the last dimension of the hidden state to zero mean / unit variance, then applies learnable scale (gamma) and shift (beta). Pre-norm placement (applied before attention and FFN) follows GPT-2's design.

CausalSelfAttention
Chapter 3 — Attention Mechanisms

Multi-head causal attention. A single qkv linear projects to Q, K, V in one shot, then splits by head. The lower-triangular mask (torch.tril) ensures no position can attend to future tokens. The buffer is registered so it moves to GPU automatically with .to(device).

FeedForward
Chapter 4 — Implementing GPT from Scratch

Position-wise FFN: expand to 4×d_model → GELU activation → contract back to d_model. The 4× expansion factor is standard from the original Transformer paper and GPT-2. GELU is smoother than ReLU and empirically trains better for language models.

TransformerBlock
Chapter 4 — Implementing GPT from Scratch

One transformer layer: pre-LayerNorm → attention → residual add, then pre-LayerNorm → FFN → residual add. Residual connections (the x = x + ... pattern) are critical for gradient flow through deep networks — without them, gradients vanish in layers beyond ~4.

MiniGPT
Chapters 4 & 5 — Full Model + Pretraining + Decoding

Token embedding + learned positional embedding → dropout → N transformer blocks → final LayerNorm → linear head. Weight tying shares parameters between tok_emb and lm_head, saving ~8K params and improving generalisation. The generate() method implements temperature sampling with optional top-k filtering.

# model.py — Mini GPT Architecture import torch import torch.nn as nn import torch.nn.functional as F import math class LayerNorm(nn.Module): def __init__(self, d_model, eps=1e-5): super().__init__() self.gamma = nn.Parameter(torch.ones(d_model)) self.beta = nn.Parameter(torch.zeros(d_model)) self.eps = eps def forward(self, x): mean = x.mean(-1, keepdim=True) std = x.std(-1, keepdim=True, unbiased=False) return self.gamma * (x - mean) / (std + self.eps) + self.beta class CausalSelfAttention(nn.Module): def __init__(self, cfg): super().__init__() d, h = cfg['d_model'], cfg['num_heads'] assert d % h == 0 self.h, self.d_k = h, d // h self.qkv = nn.Linear(d, 3 * d, bias=cfg['bias']) self.proj = nn.Linear(d, d, bias=cfg['bias']) self.attn_drop = nn.Dropout(cfg['dropout']) self.proj_drop = nn.Dropout(cfg['dropout']) # Causal mask (registered as buffer — not a parameter) T = cfg['context_length'] self.register_buffer('mask', torch.tril(torch.ones(T, T)).view(1, 1, T, T)) def forward(self, x): B, T, C = x.shape Q, K, V = self.qkv(x).split(C, dim=2) Q = Q.view(B, T, self.h, self.d_k).transpose(1, 2) K = K.view(B, T, self.h, self.d_k).transpose(1, 2) V = V.view(B, T, self.h, self.d_k).transpose(1, 2) scores = Q @ K.transpose(-2, -1) / math.sqrt(self.d_k) scores = scores.masked_fill(self.mask[:, :, :T, :T] == 0, float('-inf')) weights = self.attn_drop(F.softmax(scores, dim=-1)) out = (weights @ V).transpose(1, 2).contiguous().view(B, T, C) return self.proj_drop(self.proj(out)) class FeedForward(nn.Module): def __init__(self, cfg): super().__init__() d = cfg['d_model'] self.net = nn.Sequential( nn.Linear(d, 4 * d, bias=cfg['bias']), nn.GELU(), nn.Linear(4 * d, d, bias=cfg['bias']), nn.Dropout(cfg['dropout']) ) def forward(self, x): return self.net(x) class TransformerBlock(nn.Module): def __init__(self, cfg): super().__init__() self.ln1 = LayerNorm(cfg['d_model']) self.attn = CausalSelfAttention(cfg) self.ln2 = LayerNorm(cfg['d_model']) self.ffn = FeedForward(cfg) def forward(self, x): x = x + self.attn(self.ln1(x)) # residual + attention x = x + self.ffn(self.ln2(x)) # residual + FFN return x class MiniGPT(nn.Module): def __init__(self, cfg): super().__init__() self.cfg = cfg self.tok_emb = nn.Embedding(cfg['vocab_size'], cfg['d_model']) self.pos_emb = nn.Embedding(cfg['context_length'], cfg['d_model']) self.drop = nn.Dropout(cfg['dropout']) self.blocks = nn.Sequential(*[TransformerBlock(cfg) for _ in range(cfg['num_layers'])]) self.ln_f = LayerNorm(cfg['d_model']) self.lm_head = nn.Linear(cfg['d_model'], cfg['vocab_size'], bias=False) # Weight tying: share embedding matrix with output projection self.lm_head.weight = self.tok_emb.weight # Print parameter count n_params = sum(p.numel() for p in self.parameters()) print(f"MiniGPT: {n_params/1e6:.2f}M parameters") def forward(self, idx, targets=None): B, T = idx.shape tok = self.tok_emb(idx) pos = self.pos_emb(torch.arange(T, device=idx.device)) x = self.drop(tok + pos) x = self.blocks(x) x = self.ln_f(x) logits = self.lm_head(x) # [B, T, vocab_size] loss = None if targets is not None: loss = F.cross_entropy(logits.view(-1, logits.size(-1)), targets.view(-1)) return logits, loss @torch.no_grad() def generate(self, idx, max_new_tokens, temperature=1.0, top_k=None): for _ in range(max_new_tokens): idx_crop = idx[:, -self.cfg['context_length']:] logits, _ = self(idx_crop) logits = logits[:, -1, :] / temperature if top_k is not None: v, _ = torch.topk(logits, min(top_k, logits.size(-1))) logits[logits < v[:, -1:]] = float('-inf') probs = F.softmax(logits, dim=-1) idx_next = torch.multinomial(probs, num_samples=1) idx = torch.cat([idx, idx_next], dim=1) return idx
Weight tying (Ch 4): self.lm_head.weight = self.tok_emb.weight makes the output projection share the same matrix as the input embedding. This halves the number of parameters in the embedding table and consistently improves perplexity on language modelling benchmarks. GPT-2 uses this trick too.
Temperature & top-k (Ch 5): In generate(), dividing logits by temperature before softmax controls randomness. Temperature < 1 sharpens the distribution (more deterministic); temperature > 1 flattens it (more creative). top_k masks all but the k most probable tokens before sampling, avoiding very-low-probability "garbage" characters.

train.py — Training Loop

The training loop puts everything together. It follows the standard PyTorch pattern from Chapter 5: forward pass → cross-entropy loss → backward → gradient clip → optimizer step → periodic evaluation.

What each section does
1

Setup — load data, build model, create optimizer

Calls get_loaders() from data.py, instantiates MiniGPT, creates an AdamW optimizer with weight decay. AdamW decouples weight decay from the gradient update — it regularises parameters directly rather than through the gradient, which is the correct way to do L2 regularisation with Adam (Ch 5, Appendix D).

2

Main loop — fetch batch, forward, backward, step

Uses a persistent iterator over the DataLoader. When the iterator is exhausted (one epoch done) it restarts automatically. This gives us exactly max_iters gradient steps regardless of dataset size or batch size.

3

Gradient clipping

clip_grad_norm_(..., 1.0) caps the global L2 norm of all gradients at 1.0. Without this, rare large-loss batches can cause a single catastrophic update that destroys the model. Essential for stable training on long sequences (Ch 5).

4

Periodic evaluation & checkpointing

Every eval_interval steps, the model is set to eval() mode (disables dropout), the validation loss is computed over 20 batches, and if it's the best seen so far, the checkpoint is saved. The checkpoint includes the model weights, optimizer state, config dict, and the vocabulary — everything needed to resume training or generate text later.

Full implementation
# train.py — Training loop import torch import torch.nn.functional as F from config import GPT_CONFIG, TRAIN_CONFIG from data import get_loaders, decode from model import MiniGPT def main(): cfg = GPT_CONFIG.copy() tcfg = TRAIN_CONFIG device = tcfg['device'] print(f"Training on: {device}") # ── 1. Data ────────────────────────────────────────────────── train_loader, val_loader, stoi, itos = get_loaders(cfg, tcfg) # ── 2. Model + Optimizer ───────────────────────────────────── model = MiniGPT(cfg).to(device) optimizer = torch.optim.AdamW( model.parameters(), lr=tcfg['lr'], weight_decay=tcfg['weight_decay'] ) # ── 3. Training loop ───────────────────────────────────────── train_iter = iter(train_loader) best_val_loss = float('inf') for step in range(tcfg['max_iters']): # Fetch next batch (restart iterator if epoch ends) try: x, y = next(train_iter) except StopIteration: train_iter = iter(train_loader) x, y = next(train_iter) x, y = x.to(device), y.to(device) # Forward + loss _, loss = model(x, y) # Backward optimizer.zero_grad() loss.backward() torch.nn.utils.clip_grad_norm_(model.parameters(), tcfg['grad_clip']) optimizer.step() # ── 4. Evaluation ──────────────────────────────────────────── if step % tcfg['eval_interval'] == 0 or step == tcfg['max_iters'] - 1: model.eval() val_losses = [] for vx, vy in val_loader: vx, vy = vx.to(device), vy.to(device) with torch.no_grad(): _, vloss = model(vx, vy) val_losses.append(vloss.item()) if len(val_losses) >= 20: break val_loss = sum(val_losses) / len(val_losses) print(f"Step {step:5d} | train loss: {loss.item():.4f} | val loss: {val_loss:.4f}") # Save best checkpoint if val_loss < best_val_loss: best_val_loss = val_loss torch.save({ 'step': step, 'model_state': model.state_dict(), 'optimizer_state': optimizer.state_dict(), 'val_loss': val_loss, 'cfg': cfg, 'stoi': stoi, 'itos': itos, }, tcfg['checkpoint_path']) print(f" → Checkpoint saved (val_loss: {val_loss:.4f})") model.train() print("\nTraining complete!") if __name__ == '__main__': main()

generate.py — Load & Generate

Load the saved checkpoint and generate text by sampling from the model autoregressively. Each new character is sampled from the model's predicted probability distribution over the 65-character vocabulary.

# generate.py — Load checkpoint and generate text import torch from model import MiniGPT def load_model(checkpoint_path='checkpoint.pt'): ckpt = torch.load(checkpoint_path, map_location='cpu') model = MiniGPT(ckpt['cfg']) model.load_state_dict(ckpt['model_state']) model.eval() return model, ckpt['stoi'], ckpt['itos'] def generate_text(model, stoi, itos, prompt="ROMEO:", max_new_tokens=200, temperature=0.8, top_k=40): device = next(model.parameters()).device context = torch.tensor([[stoi.get(c, 0) for c in prompt]], dtype=torch.long, device=device) output_ids = model.generate(context, max_new_tokens, temperature=temperature, top_k=top_k) return ''.join(itos[i] for i in output_ids[0].tolist()) if __name__ == '__main__': print("Loading model...") model, stoi, itos = load_model() print(f"Model loaded. Generating text...\n{'='*50}") prompts = ["ROMEO:", "HAMLET:", "To be or not to be,"] for p in prompts: print(f"\nPrompt: {p}") print(generate_text(model, stoi, itos, prompt=p)) print("-" * 40)

What's Next — Scale Up & Go Further

You have a working mini GPT. Here are the natural next steps to go from this toy project to something production-grade.

1. Train on custom text

Replace input.txt with any .txt file — song lyrics, a novel, code, emails. The only change needed is deleting input.txt before running.

2. Scale up the model

In config.py set d_model=256, num_layers=6, context_length=128. You'll need a GPU — this is how you go from 0.8M to ~10M parameters.

3. Add BPE tokenization

Replace CharDataset with tiktoken.get_encoding("gpt2"). Bump vocab_size to 50,257. This matches GPT-2's actual tokenizer (Ch 2).

4. Add instruction fine-tuning

Format data as prompt/response pairs using Alpaca format, mask the prompt tokens in the loss (set to -100), fine-tune on Q&A data. This is Chapter 7 applied directly.

5. Add LoRA

Replace the qkv and proj linears in CausalSelfAttention with LoRALinear from Appendix E. Freeze base weights, train only the low-rank matrices. 10–100× fewer trainable params.

6. Deploy as a web app

Export the model to ONNX (torch.onnx.export), serve with FastAPI or Gradio, or run inference in-browser with ONNX Runtime Web. No GPU server needed for the mini model.

Every one of these extensions is covered in detail in Build a Large Language Model (From Scratch) by Sebastian Raschka. This capstone is your bridge from reading to building.
