Chapter 3

Coding Attention Mechanisms

7 Lessons 8-Question Quiz ~40 min read Core Architecture

Attention is the single most important architectural innovation in modern language models. It is what allows a transformer to capture long-range dependencies in a sequence, understand what each word refers to, and build rich contextual representations — all in a single forward pass. This chapter builds attention from first principles: starting from a simple intuition, deriving the scaled dot-product formula, adding learnable projections, enforcing causal masking, and finally scaling up to multi-head attention as used in GPT-2.

3.1

The Problem with Long Sequences

The first sequence-to-sequence (seq2seq) neural networks, used for tasks like machine translation, followed an encoder-decoder RNN architecture. The encoder read the full input sentence token by token, updating a hidden state at every step. When it finished, that final hidden state — a single fixed-size vector — was passed to the decoder as the "context". The decoder then generated the output translation, one token at a time, using only this single context vector.

This design has a fundamental weakness: the information bottleneck. Every piece of information in the input sentence — all its words, their relationships, the grammar — must be compressed into a single fixed-size vector. For short sentences this is manageable, but for long sequences (say, 50+ words), the early parts of the sentence are essentially forgotten by the time the encoder reaches the end. The vector simply cannot hold everything. Performance degrades noticeably as input length increases.

Attention was invented to break this bottleneck. Instead of forcing the decoder to rely on a single compressed context vector, attention gives each output position a direct, weighted read over all encoder hidden states. When generating the translation of word i, the decoder can "attend" strongly to the relevant input words and weakly to irrelevant ones — effectively computing a new, position-specific context vector for every output step.

GPT uses self-attention, which is different from the original encoder-decoder attention. In self-attention, there is no separate encoder and decoder — a single stack of transformer blocks processes the sequence. Each token attends to all other tokens in the same sequence, building rich contextual representations for every position simultaneously. This is far more powerful than RNNs and, crucially, fully parallelizable: all positions are processed in one matrix operation rather than token-by-token.

Key Contrast: In encoder-decoder attention (used in translation), each decoder token attends to encoder outputs. In GPT-style self-attention, each token in a single sequence attends to all other tokens in that same sequence. The goal is not translation — it is building context-aware representations to predict the next token.

3.2

Self-Attention Intuition

Self-attention is best understood through a concrete example. Consider the sentence: "The animal didn't cross the street because it was too tired." When a human reads "it", they immediately understand it refers to "animal" — not "street". But how does the model figure this out?

Self-attention computes, for each token, a score representing how relevant every other token is to interpreting its meaning. When processing the token "it", the attention mechanism assigns a high weight to "animal" (because "it" is a pronoun that grammatically relates to the noun subject) and a low weight to "street" (even though it is another noun, the context makes "animal" the better referent). The output representation of "it" is then a weighted combination of all token representations, dominated by "animal".

This happens simultaneously for every token in the sequence. When processing "tired", the model might attend heavily to "it" and "animal". When processing "cross", it might attend to "street" and "didn't". Each token builds its own context-specific representation by selectively aggregating information from the rest of the sequence.

The Core Question: Self-attention is the model asking — "Given what I am (this token), which other tokens in the sequence are most relevant to understanding my meaning?" The answer is a probability distribution over all positions, which is used to compute a weighted sum.

3.3

Simplified Self-Attention (No Learned Weights)

Before adding trainable parameters, let us understand the raw mechanism. In simplified self-attention, we compute attention directly between the input token embeddings themselves — no projections, no extra parameters. This gives us a clean view of the three-step procedure that is at the heart of all attention variants.

1
Compute raw attention scores (dot products)

For each pair of tokens (i, j), compute the dot product of their embedding vectors: score(i,j) = xi · xj. This gives an unnormalised measure of "similarity" between token i and token j. Higher dot product = more similar direction in embedding space = higher relevance. The result is an [N×N] score matrix where N is the sequence length.

2
Normalise with softmax to get attention weights

Apply softmax row-wise: weights(i, :) = softmax(scores(i, :]). This converts raw scores into a proper probability distribution — all weights in each row sum to exactly 1 and are non-negative. Now each row describes "how much token i attends to every other token".

3
Compute context vectors (weighted sum)

For each token i, compute a new "context-aware" representation: context(i) = Σj weights(i,j) × xj. This is a weighted average of all token embeddings, where the weights determine how much each token contributes. Token i ends up with a representation that blends in information from the tokens it attends to most.

This is the complete pipeline: scores = X @ X.T, weights = softmax(scores), context = weights @ X. Three matrix operations, no learned parameters.

import torch
import torch.nn.functional as F

# 3 tokens, 3-dim embeddings
x = torch.tensor([[0.43, 0.15, 0.89],   # token 1
                   [0.55, 0.87, 0.66],   # token 2
                   [0.57, 0.85, 0.64]])  # token 3

# Step 1: raw attention scores (unnormalised)
scores  = x @ x.T                    # shape [3, 3]

# Step 2: softmax to get attention weights summing to 1
weights = F.softmax(scores, dim=-1) # shape [3, 3]

# Step 3: weighted sum of value vectors
context = weights @ x                # shape [3, 3]

print(weights.sum(dim=-1))  # tensor([1., 1., 1.]) — rows sum to 1
Softmax converts unconstrained dot-product scores into a probability distribution over positions. The resulting attention weights define a convex combination of the value vectors — the context vector is literally a weighted average of all token embeddings in the sequence.
3.4

Trainable QKV Projections

The simplified version above uses the raw token embeddings to compute attention — but the embeddings were optimized for a different purpose (representing semantic meaning), not for finding which tokens are relevant to which. The improved design introduces three separate learned linear projections: WQ, WK, WV, each of shape [d_model, d_k].

Each token embedding xi is projected into three separate vectors:

  • Query (qi = xi WQ) — what this token is looking for
  • Key (ki = xi WK) — what this token has to offer for matching
  • Value (vi = xi WV) — the actual content to retrieve when matched

The attention formula becomes: Attention(Q, K, V) = softmax(Q @ K.T / √d_k) @ V. The √d_k scaling factor is critical: for large d_k, dot products grow in magnitude proportionally to d_k, pushing softmax into regions with near-zero gradients (the softmax becomes a one-hot distribution). Dividing by √d_k keeps the variance of the dot products roughly constant regardless of dimensionality, preserving healthy gradients during training.

Database Analogy: Think of attention as a soft database lookup. The Query is "what I'm searching for". The Keys are "what the database has indexed". Matching Q against K produces attention weights — how relevant each entry is. The Values are the actual data returned when a key matches your query. Unlike a hard database (exact match), attention is differentiable and soft — every entry contributes proportionally to its match score.

import torch.nn as nn
import torch.nn.functional as F

class SelfAttention(nn.Module):
    def __init__(self, d_in, d_out):
        super().__init__()
        self.W_q = nn.Linear(d_in, d_out, bias=False)
        self.W_k = nn.Linear(d_in, d_out, bias=False)
        self.W_v = nn.Linear(d_in, d_out, bias=False)

    def forward(self, x):
        Q = self.W_q(x)   # [B, T, d_out]
        K = self.W_k(x)   # [B, T, d_out]
        V = self.W_v(x)   # [B, T, d_out]
        # Scaled dot-product attention
        scores  = Q @ K.transpose(-2, -1) / K.shape[-1]**0.5
        weights = F.softmax(scores, dim=-1)
        return weights @ V  # [B, T, d_out]
The three weight matrices WQ, WK, WV are learned during training. Separating Query, Key, and Value allows the model to independently learn "what to search for", "what to match against", and "what to return" — giving far more representational flexibility than using the raw embeddings directly.
3.5

Causal (Masked) Self-Attention

GPT is an autoregressive language model: it generates text one token at a time, left to right. At each step, it can only use tokens that have already been generated — it must never "peek" at future tokens. During training, this constraint must be enforced, because if the model can see the answer (future tokens) while computing attention, it learns nothing useful.

This is solved with a causal mask, also called a lower-triangular mask. Before computing the softmax, we set all positions in the attention score matrix where j > i (future positions) to -∞. When softmax is applied, exp(-∞) = 0, so those positions receive exactly zero attention weight. Token i can attend to tokens 0, 1, …, i (itself and all past tokens) but receives zero contribution from tokens i+1, i+2, … onwards.

The mask pattern for a 4-token sequence looks like a lower triangle:

Pos → 0 1 2 3
Token 0 ✓ −∞ −∞ −∞
Token 1 ✓ ✓ −∞ −∞
Token 2 ✓ ✓ ✓ −∞
Token 3 ✓ ✓ ✓ ✓

T = x.shape[1]  # sequence length

# Create upper-triangular boolean mask (True = should be masked)
mask = torch.triu(torch.ones(T, T), diagonal=1).bool()

# Set masked positions to -inf before softmax
scores = scores.masked_fill(mask, float('-inf'))
weights = F.softmax(scores, dim=-1)
# exp(-inf) = 0, so masked positions get zero attention weight
The causal mask is a one-time operation per forward pass. torch.triu(ones, diagonal=1) creates an upper-triangular matrix; masked_fill sets those positions to −∞. After softmax, these become exactly 0, ensuring no information from future tokens leaks into past token representations.
3.6

Multi-Head Attention

A single attention head learns one "type" of relationship in the data. But language has many simultaneously active relationships: token i might have a syntactic dependency on token j, a coreference link to token k, and a semantic similarity to token l — all at the same time. A single attention head's weight matrix can only attend to one combination of these patterns at once.

Multi-head attention solves this by running h attention heads in parallel, each with its own independent WQ, WK, WV projection matrices. Each head operates on a lower-dimensional subspace (d_k = d_model / h), so the total computation is the same as a single attention operation. The outputs of all heads are then concatenated and projected through an output matrix WO.

In practice, the heads learn complementary specializations:

  • Head 1 might track syntactic subject-verb dependencies
  • Head 2 might resolve pronoun coreferences
  • Head 3 might capture semantic role relationships
  • Head 4 might attend to positional proximity

GPT-2 Small stats: d_model = 768, num_heads = 12, so d_k = 768 / 12 = 64 per head. Total parameters per multi-head attention block = 4 × d_model × d_model = 4 × 768 × 768 = 2,359,296 (one each for WQ, WK, WV, WO).

After all heads run in parallel, their outputs are concatenated and linearly projected.
class MultiHeadAttention(nn.Module):
    def __init__(self, d_model, num_heads):
        super().__init__()
        assert d_model % num_heads == 0
        self.h    = num_heads
        self.d_k  = d_model // num_heads
        self.W_q  = nn.Linear(d_model, d_model)
        self.W_k  = nn.Linear(d_model, d_model)
        self.W_v  = nn.Linear(d_model, d_model)
        self.W_o  = nn.Linear(d_model, d_model)

    def forward(self, x):
        B, T, C = x.shape
        # Project & split into h heads: [B, T, C] → [B, h, T, d_k]
        Q = self.W_q(x).view(B, T, self.h, self.d_k).transpose(1, 2)
        K = self.W_k(x).view(B, T, self.h, self.d_k).transpose(1, 2)
        V = self.W_v(x).view(B, T, self.h, self.d_k).transpose(1, 2)
        # Attention for all heads in parallel
        scores = Q @ K.transpose(-2, -1) / self.d_k**0.5
        out    = F.softmax(scores, dim=-1) @ V
        # Concatenate heads: [B, h, T, d_k] → [B, T, C]
        out = out.transpose(1, 2).contiguous().view(B, T, C)
        return self.W_o(out)  # Final output projection
3.7

Dropout in Attention

Like other components of a deep neural network, the attention mechanism is susceptible to overfitting: the model may learn to always heavily attend to the same specific token pairs in the training data, rather than generalising to flexible, content-dependent patterns. This is especially risky in attention because the soft alignment weights can become very peaked (near one-hot) — the model "locks in" to specific token relationships that may not generalise.

Dropout combats this by randomly zeroing out some attention weights during training, with probability p (typically 0.1 to 0.2). At each training step, a different random subset of attention connections is disabled. The model must therefore learn to not over-rely on any single attention path — it must develop multiple, redundant pathways to represent each relationship. This acts as a form of regularisation that improves generalisation.

In GPT-2, dropout is applied at two points in the attention block:

  1. After the softmax attention weights — randomly zeroing some weights before the weighted sum with V
  2. After the output projection WO — applied to the final attention output before adding to the residual stream

At inference time, dropout is disabled (by calling model.eval()), and all attention weights are used — the model has already learned robust patterns during training that do not over-rely on any single connection.

class MultiHeadAttentionWithDropout(nn.Module):
    def __init__(self, d_model, num_heads, dropout=0.1):
        super().__init__()
        self.attn_drop = nn.Dropout(dropout)  # on attention weights
        self.proj_drop = nn.Dropout(dropout)  # on output projection
        # ... W_q, W_k, W_v, W_o as before ...

    def forward(self, x):
        # ... compute Q, K, V as before ...
        scores  = Q @ K.transpose(-2,-1) / self.d_k**0.5
        weights = F.softmax(scores, dim=-1)
        weights = self.attn_drop(weights)  # ← dropout on weights
        out     = weights @ V
        # ... reshape and project ...
        return self.proj_drop(self.W_o(out))  # ← dropout on output
Dropout is only active during training (model.train()). During inference (model.eval()), PyTorch automatically disables all dropout layers. The p=0.1 setting in GPT-2 means roughly 10% of attention weights are zeroed per training step — light regularisation to prevent overfitting without significantly disrupting learning.

Chapter 3 — Key Takeaways

  • Self-attention computes context-aware representations by allowing every token to attend to all other tokens simultaneously — breaking the information bottleneck of RNN seq2seq models.
  • QKV projections (WQ, WK, WV) are three separate learned linear transformations that independently learn what to query, what to match against, and what to retrieve — giving far more representational power than using raw embeddings.
  • Scaling by √d_k prevents the dot products from growing too large as embedding dimensions increase, keeping the softmax distribution well-behaved and gradients healthy during training.
  • Causal masking (lower-triangular mask) ensures GPT never attends to future tokens — preserving the autoregressive property required for left-to-right text generation. Future positions are set to −∞ before softmax.
  • Multi-head attention runs h attention heads in parallel, each with independent projections, allowing different heads to specialise in different relationship types (syntactic, semantic, coreference, etc.).
  • Dropout randomly zeros attention weights during training (p=0.1–0.2) to prevent the model from over-relying on specific token pairs — acting as regularisation that improves generalisation.
