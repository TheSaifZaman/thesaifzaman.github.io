Appendix D

Training Loop Extras

3 Lessons 5-Question Quiz ~15 min read LR Warmup Cosine Decay Gradient Clipping

The core training loop — forward, loss, backward, step — works for toy models, but production LLM training requires three additional techniques to be stable and efficient. Learning rate warmup prevents catastrophic early updates. Cosine learning rate decay ensures the model converges smoothly. Gradient clipping prevents the occasional large gradient spikes from transformer attention from destabilising training. Together, these three techniques form the standard training recipe used by GPT-3, LLaMA, and virtually every large language model trained since 2020.

Understanding why each technique is needed — rather than just using them mechanically — lets you debug training instabilities when they arise. A loss spike followed by recovery usually indicates a gradient clipping threshold that is too high. A model that trains well early but plateaus without improving typically needs better learning rate scheduling. A model that diverges immediately from step 1 almost certainly needs warmup. These patterns are worth recognising.

D.1

Learning Rate Warmup

Learning rate warmup addresses a fundamental instability in neural network training: at step 0, all model weights are randomly initialised. The gradients computed from this random initialisation can be large and point in conflicting directions across different layers — the model has no useful signal yet. If you start training with a large learning rate (e.g., 6e-4, typical for GPT-2), the first few gradient steps can take enormous strides in parameter space and land in a poor region that is hard to recover from. The model may even diverge immediately.

The solution is linear warmup: start the learning rate at 0 (or a very small value), and increase it linearly over the first warmup_steps training steps until it reaches the target maximum. During warmup, the model makes cautious, small updates. By the time the LR reaches its maximum value, the weights have moved away from their random initialisation into a region where the loss landscape is more predictable and large steps are safer. GPT-3 uses 375 million token warmup (375 steps with batch size 1 million tokens); the original Transformer paper uses 4000 steps. The exact length is a hyperparameter, but a range of 1–5% of total training steps is typical.

The original Transformer paper used an elegant combined schedule: linear warmup followed by inverse-square-root decay. The formula d_model^-0.5 × min(step^-0.5, step × warmup_steps^-1.5) increases like a ramp during warmup and then decays like 1/√step afterward. Modern LLMs typically use this warmup phase combined with cosine decay rather than inverse-square-root decay — cosine decay is smoother and allows tuning the final LR explicitly.

def get_lr(step, d_model, warmup_steps=4000):
    # Original Transformer paper schedule
    arg1 = step ** -0.5
    arg2 = step * (warmup_steps ** -1.5)
    return d_model ** -0.5 * min(arg1, arg2)

scheduler = torch.optim.lr_scheduler.LambdaLR(
    optimizer,
    lr_lambda=lambda step: get_lr(step+1, d_model=768)
)

D.2

Cosine Learning Rate Decay

After the warmup phase, the learning rate should decrease over training — but how? Step decay (dropping LR by a fixed factor at predefined steps) creates sharp discontinuities that can cause loss spikes. Exponential decay decreases too rapidly early and too slowly late. Cosine decay is the solution: the learning rate follows a half-cosine curve from max_lr down to min_lr (typically 10% of max_lr). The cosine function decreases slowly at the extremes and faster in the middle, which happens to match the natural pace of neural network training — fast progress early, slowing as the model approaches its optimum.

Cosine decay has two key properties that make it the dominant schedule for LLM training. First, it is smooth — no sharp drops means no sudden changes in the gradient dynamics. Second, it is predictable — given the total number of training steps, you know exactly where the LR will be at any point. This allows you to plan training budgets and stop-and-resume checkpointing without perturbing the schedule. GPT-3 uses cosine decay from max_lr=6e-4 to min_lr=6e-5 (exactly 10%) over the entire training run. LLaMA uses the same structure. It has become the default because it consistently performs well across a wide range of model sizes and datasets.

A subtlety worth noting: the min_lr floor matters. Decaying to 0 would mean the model stops learning entirely at the end of training — useful if you want the model to fully converge. But setting min_lr to 10% of max keeps a small amount of learning happening until the very end, which helps with the long tail of rare training examples. The exact ratio is an empirical finding rather than a theoretical result, and 10% is widely used without strong justification beyond "it works well."

import math

def cosine_lr(step, warmup_steps, max_steps, max_lr, min_lr):
    # Linear warmup phase
    if step < warmup_steps:
        return max_lr * step / warmup_steps
    # Cosine decay phase
    progress = (step - warmup_steps) / (max_steps - warmup_steps)
    return min_lr + 0.5 * (max_lr - min_lr) * (1 + math.cos(math.pi * progress))

# Example: max_lr=6e-4, min_lr=6e-5, warmup=100 steps, total=10000
for step in range(10000):
    lr = cosine_lr(step, 100, 10000, 6e-4, 6e-5)
    for param_group in optimizer.param_groups:
        param_group['lr'] = lr
Warmup + Cosine Decay Combined Schedule
Linear warmup ramps from 0 to max_lr. Cosine decay then smoothly reduces LR to min_lr (10% of max). No sharp discontinuities — training remains stable throughout.
D.3

Gradient Clipping

Gradient clipping addresses a different problem from learning rate scheduling: occasional catastrophic gradient spikes. In transformer training, the attention mechanism can sometimes produce very large attention weights — if a single key-query pair has an extremely high similarity score, the softmax concentrates nearly all weight on that pair. When this concentration is very sharp, the gradient through the softmax with respect to the keys and queries can be enormous. One such spike can update the weights by a large amount in a random direction, destroying previously learned representations and causing the loss to jump upward — a training instability that can be hard to recover from.

Gradient clipping prevents this by monitoring the global gradient norm: the Euclidean norm across all parameter gradients treated as one long vector. If this norm exceeds a threshold (typically 1.0), all gradients are scaled down by the factor threshold / norm. Critically, all gradients are scaled by the same factor, so the direction of the update vector is preserved — only its magnitude is capped. This is fundamentally different from per-parameter gradient clamping, which would distort the direction of the update.

In PyTorch, gradient clipping is a single line: torch.nn.utils.clip_grad_norm_(model.parameters(), max_norm=1.0). It must be called after loss.backward() (which populates the gradients) and before optimizer.step() (which uses them). The threshold of 1.0 is the standard for transformer training; values like 0.1 are overly aggressive and slow convergence, while 10.0 is too permissive to prevent spikes. If you find that gradient clipping triggers very frequently (you can monitor the norm), it is a signal that the learning rate may be too high or the model may have numerical issues.

for x, y in dataloader:
    optimizer.zero_grad()
    loss = F.cross_entropy(model(x), y)
    loss.backward()                            # 1. compute gradients

    # 2. clip gradients (BEFORE optimizer.step)
    grad_norm = torch.nn.utils.clip_grad_norm_(
        model.parameters(), max_norm=1.0
    )
    # grad_norm is the UNCLIPPED norm (useful for logging)

    optimizer.step()                            # 3. update weights

The Standard LLM Training Recipe
1 Linear LR warmup for the first 1–5% of training steps (ramp from 0 → max_lr)
2 Cosine LR decay from max_lr down to min_lr (10% of max) over the remaining steps
3 Gradient clipping with max_norm=1.0 at every training step

Appendix D — Key Takeaways

  • Learning rate warmup prevents unstable early updates when weights are randomly initialised. Linear warmup from 0 to max_lr over the first 1–5% of training steps is standard.
  • Cosine decay smoothly reduces the LR from max_lr to min_lr (typically 10% of max) following a half-cosine curve. Used by GPT-3, LLaMA, and most modern LLMs.
  • Gradient clipping (max_norm=1.0) caps the global gradient norm to prevent catastrophic spikes from transformer attention. It preserves gradient direction — only magnitude is scaled.
  • Gradient clipping must be called after loss.backward() and before optimizer.step(). Calling it in the wrong order has no effect.
  • The standard recipe: warmup + cosine decay + clipping is stable across model sizes from 100M to 100B parameters. It is not the only valid recipe, but it is the safe default.
