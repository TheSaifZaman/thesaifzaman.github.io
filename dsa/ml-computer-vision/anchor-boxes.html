Anchor Boxes

Medium Time: O(n·k) Space: O(n·k)

How It Works

Anchor boxes (also called prior boxes or default boxes) are predefined bounding boxes of various scales and aspect ratios placed at each spatial location in a feature map. They serve as reference templates that the detector refines to predict actual object locations.

Instead of predicting boxes from scratch, the network predicts offsets (dx, dy, dw, dh) relative to each anchor, plus a confidence score. This makes detection much more stable and efficient. Faster R-CNN, SSD, and YOLO all use anchor-based approaches.

Key Properties

  • Scales: Multiple sizes (e.g., 32, 64, 128, 256, 512 pixels).
  • Aspect ratios: Typically 1:1, 1:2, 2:1 per scale.
  • Grid-based: One set of anchors per spatial location in feature map.
  • Matching: Anchors matched to ground truth by IoU for training.

Implementation

function generateAnchors(gridSize, scales, ratios) {
    var anchors = [];
    var cellSize = 1.0 / gridSize;
    for (var r = 0; r < gridSize; r++) {
        for (var c = 0; c < gridSize; c++) {
            var cx = (c + 0.5) * cellSize;
            var cy = (r + 0.5) * cellSize;
            for (var s = 0; s < scales.length; s++) {
                for (var a = 0; a < ratios.length; a++) {
                    var w = scales[s] * Math.sqrt(ratios[a]);
                    var h = scales[s] / Math.sqrt(ratios[a]);
                    anchors.push({cx: cx, cy: cy, w: w, h: h});
                }
            }
        }
    }
    return anchors;
}
