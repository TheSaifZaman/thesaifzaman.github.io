Histogram of Oriented Gradients

Medium Time: O(n·m) Space: O(n·m)

How It Works

HOG divides the image into small cells, computes gradient direction histograms within each cell, and normalizes them over larger blocks. The resulting feature vector captures the distribution of local edge orientations, making it highly effective for object detection (especially pedestrians).

Gradient directions are quantized into bins (typically 9 bins of 20° each covering 0°–180°). Each pixel votes for its gradient direction bin, weighted by the gradient magnitude. Block normalization reduces sensitivity to illumination changes.

Key Properties

  • Cell size: Typically 8×8 pixels per cell.
  • Block size: 2×2 cells with overlapping for normalization.
  • Bins: 9 orientation bins (unsigned gradients: 0°–180°).
  • Normalization: L2-norm across blocks for illumination invariance.

Implementation

function computeHOG(image, cellSize, numBins) {
    var rows = image.length, cols = image[0].length;
    var binWidth = 180 / numBins;
    var cells = [];
    for (var cy = 0; cy < rows; cy += cellSize) {
        var cellRow = [];
        for (var cx = 0; cx < cols; cx += cellSize) {
            var hist = new Array(numBins).fill(0);
            for (var r = cy; r < cy + cellSize && r < rows - 1; r++) {
                for (var c = cx; c < cx + cellSize && c < cols - 1; c++) {
                    var gx = image[r][c + 1] - image[r][c];
                    var gy = image[r + 1][c] - image[r][c];
                    var mag = Math.sqrt(gx * gx + gy * gy);
                    var angle = (Math.atan2(gy, gx) * 180 / Math.PI + 180) % 180;
                    var bin = Math.floor(angle / binWidth) % numBins;
                    hist[bin] += mag;
                }
            }
            cellRow.push(hist);
        }
        cells.push(cellRow);
    }
    return cells;
}
