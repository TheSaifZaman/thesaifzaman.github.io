Beam Search

Medium Time: O(b·V·T) Space: O(b·T)

How It Works

Beam Search is a heuristic search algorithm used in sequence decoding (machine translation, text generation, speech recognition). At each time step, instead of keeping only the single best candidate (greedy) or all candidates (exhaustive), it keeps the top b candidates, where b is the beam width.

At each step: (1) expand each of the b candidates by all possible next tokens, (2) score all expansions, (3) keep only the top b. This explores a wider space than greedy search while remaining tractable compared to exhaustive search.

Beam width b=1 is greedy search. Larger b finds better sequences but costs more compute. Typical values are b=4 to b=10 in practice.

Top-k Expansion

Keeps b best candidates at each decoding step.

Quality vs Speed

Larger beam width = better results but slower decoding.

Translation

Standard decoding method for seq2seq and transformer models.

Length Penalty

Normalizes scores by length to avoid preferring short sequences.

Implementation

function beamSearch(model, startToken, beamWidth, maxLen) {
    var beams = [{ tokens: [startToken], score: 0 }];

    for (var t = 0; t < maxLen; t++) {
        var candidates = [];
        for (var i = 0; i < beams.length; i++) {
            var beam = beams[i];
            if (beam.tokens[beam.tokens.length - 1] === EOS) {
                candidates.push(beam); // keep finished beams
                continue;
            }
            // Get next token probabilities
            var logProbs = model.predict(beam.tokens);
            for (var v = 0; v < logProbs.length; v++) {
                candidates.push({
                    tokens: beam.tokens.concat([v]),
                    score: beam.score + logProbs[v]
                });
            }
        }
        // Keep top beamWidth candidates
        candidates.sort(function(a, b) { return b.score - a.score; });
        beams = candidates.slice(0, beamWidth);
    }
    return beams[0].tokens; // best sequence
}
