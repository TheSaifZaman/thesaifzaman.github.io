Apriori Algorithm

Medium Time: O(2d) Space: O(2d)

How It Works

The Apriori algorithm discovers frequent itemsets in transactional data for association rule mining. It exploits the downward closure property: if an itemset is infrequent, all its supersets must also be infrequent. This allows aggressive pruning of the search space.

The algorithm iterates: (1) generate candidate k-itemsets from frequent (k-1)-itemsets, (2) scan transactions to count support, (3) prune candidates below the minimum support threshold. Rules like "bread & butter ⇒ milk" are extracted from frequent itemsets.

Key Properties

  • Support: Fraction of transactions containing an itemset.
  • Confidence: P(B|A) = support(A∪B) / support(A).
  • Lift: confidence(A⇒B) / support(B). Lift > 1 means positive association.
  • Anti-monotone: Subsets of frequent sets are always frequent.

Implementation

function apriori(transactions, minSupport) {
    var n = transactions.length;
    // Count 1-itemsets
    var counts = {};
    for (var t = 0; t < n; t++) {
        for (var i = 0; i < transactions[t].length; i++) {
            var item = transactions[t][i];
            counts[item] = (counts[item] || 0) + 1;
        }
    }
    // Filter by min support
    var frequent = [];
    var keys = Object.keys(counts);
    for (var k = 0; k < keys.length; k++) {
        if (counts[keys[k]] / n >= minSupport) {
            frequent.push([keys[k]]);
        }
    }
    var allFrequent = frequent.slice();
    // Generate candidates and repeat
    while (frequent.length > 0) {
        var candidates = generateCandidates(frequent);
        frequent = filterBySupport(candidates, transactions, minSupport);
        allFrequent = allFrequent.concat(frequent);
    }
    return allFrequent;
}
