FP-Growth

Medium Time: O(n·m) Space: O(n·m)

How It Works

FP-Growth (Frequent Pattern Growth) mines frequent itemsets without candidate generation. It compresses the transaction database into a compact FP-tree, then extracts frequent patterns by building conditional FP-trees recursively.

The algorithm requires only two database scans: one to count item frequencies and another to build the FP-tree. Items are sorted by frequency and inserted as branches, with shared prefixes merged. This makes FP-Growth significantly faster than Apriori for dense datasets.

Key Properties

  • No candidates: Avoids expensive candidate generation and testing.
  • Compact: FP-tree compresses the database by sharing prefixes.
  • Two scans: Only needs two passes over the database.
  • Divide & conquer: Recursively mines conditional trees for each item.

Implementation

function buildFPTree(transactions, minSupport) {
    // Count item frequencies
    var freq = {};
    for (var t = 0; t < transactions.length; t++) {
        for (var i = 0; i < transactions[t].length; i++) {
            freq[transactions[t][i]] = (freq[transactions[t][i]] || 0) + 1;
        }
    }
    // Filter and sort items by frequency
    var root = {item: null, count: 0, children: {}};
    for (var t2 = 0; t2 < transactions.length; t2++) {
        var sorted = transactions[t2]
            .filter(function(item) { return freq[item] >= minSupport; })
            .sort(function(a, b) { return freq[b] - freq[a]; });
        insertTree(root, sorted);
    }
    return root;
}

function insertTree(node, items) {
    if (items.length === 0) return;
    var item = items[0];
    if (!node.children[item]) {
        node.children[item] = {item: item, count: 0, children: {}};
    }
    node.children[item].count++;
    insertTree(node.children[item], items.slice(1));
}
