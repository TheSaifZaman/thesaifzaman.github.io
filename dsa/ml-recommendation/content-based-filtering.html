Content-Based Filtering

Medium Time: O(n·d) Space: O(n·d)

How It Works

Content-based filtering recommends items similar to those a user has previously liked, based on item features (genre, keywords, attributes). Each item is represented as a feature vector, and similarity (typically cosine) is computed between items to find the best matches.

Unlike collaborative filtering, it does not need other users' data and can recommend new items immediately if their features are known. However, it tends to over-specialize, only recommending items similar to what the user already knows (the "filter bubble" problem).

Key Properties

  • Feature vectors: TF-IDF, embeddings, or handcrafted item attributes.
  • User profile: Weighted average of features from liked items.
  • No cold-start for items: New items can be recommended immediately.
  • Limitation: Cannot discover items outside user's existing taste profile.

Implementation

function contentBasedRecommend(userProfile, items, topK) {
    var scores = [];
    for (var i = 0; i < items.length; i++) {
        var sim = cosineSimilarity(userProfile, items[i].features);
        scores.push({item: items[i], score: sim});
    }
    scores.sort(function(a, b) { return b.score - a.score; });
    return scores.slice(0, topK);
}

function cosineSimilarity(a, b) {
    var dot = 0, normA = 0, normB = 0;
    for (var i = 0; i < a.length; i++) {
        dot += a[i] * b[i];
        normA += a[i] * a[i];
        normB += b[i] * b[i];
    }
    return dot / (Math.sqrt(normA) * Math.sqrt(normB));
}
