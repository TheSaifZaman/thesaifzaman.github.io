Fast Exponentiation

Easy Time: O(log n) Space: O(1)

How It Works

Fast exponentiation (also called binary exponentiation or exponentiation by squaring) computes a^n in O(log n) multiplications instead of the naive O(n) approach. The idea is based on the binary representation of the exponent: we repeatedly square the base and multiply it into the result whenever the corresponding bit of the exponent is set.

For example, to compute 3^13: since 13 = 1101 in binary, we compute 3^1 * 3^4 * 3^8 = 3 * 81 * 6561 = 1594323. At each step we square the base (3 -> 9 -> 81 -> 6561) and multiply into the result when the current bit is 1.

This technique is essential in modular arithmetic (computing a^n mod m), which appears frequently in competitive programming, cryptography (RSA), and computing large Fibonacci numbers via matrix exponentiation. Combined with modular arithmetic, it handles astronomically large exponents efficiently.

Binary Decomposition
Express the exponent in binary and process each bit from least to most significant.
Square and Multiply
Square the base at each step. If the current bit is 1, multiply the result by the current base.
Modular Version
Take mod at each multiplication to prevent overflow: (a * b) mod m = ((a mod m) * (b mod m)) mod m.
O(log n) Operations
Only log2(n) multiplications needed, making it feasible for exponents up to 10^18.

Implementation

// Binary exponentiation: compute base^exp
function fastPow(base, exp) {
    var result = 1;
    while (exp > 0) {
        if (exp % 2 === 1) {
            result *= base;
        }
        base *= base;
        exp = Math.floor(exp / 2);
    }
    return result;
}

// Modular exponentiation: compute base^exp mod m
function modPow(base, exp, mod) {
    var result = 1;
    base = base % mod;
    while (exp > 0) {
        if (exp % 2 === 1) {
            result = (result * base) % mod;
        }
        exp = Math.floor(exp / 2);
        base = (base * base) % mod;
    }
    return result;
}
