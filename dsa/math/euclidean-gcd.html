Euclidean GCD

Easy Time: O(log min(a,b)) Space: O(1)

How It Works

The Euclidean algorithm is one of the oldest known algorithms, dating back to around 300 BC. It efficiently computes the greatest common divisor (GCD) of two integers. The key insight is that GCD(a, b) = GCD(b, a mod b), which allows us to repeatedly reduce the problem until one of the numbers becomes zero.

At each step, we replace the larger number with the remainder of dividing the larger by the smaller. This process converges quickly because the remainder is always strictly smaller than the divisor. When the remainder reaches zero, the other number is the GCD. The algorithm runs in O(log min(a,b)) time, making it extremely efficient even for very large numbers.

The Extended Euclidean Algorithm also finds integers x and y such that ax + by = GCD(a,b). This is crucial for computing modular inverses, which are needed in modular division, RSA cryptography, and many competitive programming problems.

Base Case
GCD(a, 0) = a. When one number is zero, the other is the GCD.
Reduction Step
GCD(a, b) = GCD(b, a % b). The remainder is always smaller than b.
LCM Connection
LCM(a, b) = a * b / GCD(a, b). Always compute GCD first to avoid overflow.
Extended GCD
Finds x, y where ax + by = GCD(a,b). Essential for modular inverses.

Implementation

// Iterative Euclidean GCD
function gcd(a, b) {
    while (b !== 0) {
        var t = b;
        b = a % b;
        a = t;
    }
    return a;
}

// Extended Euclidean Algorithm
// Returns [gcd, x, y] where a*x + b*y = gcd
function extendedGcd(a, b) {
    if (b === 0) return [a, 1, 0];
    var result = extendedGcd(b, a % b);
    var g = result[0], x1 = result[1], y1 = result[2];
    return [g, y1, x1 - Math.floor(a / b) * y1];
}

// LCM using GCD
function lcm(a, b) {
    return (a / gcd(a, b)) * b;
}
