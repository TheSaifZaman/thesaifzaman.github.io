SARSA

Medium Time: O(S·A·episodes) Space: O(S·A)

How It Works

SARSA (State-Action-Reward-State-Action) is an on-policy reinforcement learning algorithm. Unlike Q-Learning which uses the max Q-value of the next state, SARSA updates using the Q-value of the action actually taken in the next state:

Q(s, a) ← Q(s, a) + α [r + γ Q(s', a') - Q(s, a)]

Because SARSA follows and improves the policy it is actually using (including exploration), it tends to learn safer, more conservative paths. Near cliffs or penalties, SARSA avoids risky edges that Q-Learning might exploit, making it better suited for situations where the exploration policy matters during training.

On-Policy

Updates Q using the action the agent actually takes next.

Conservative

Learns safer paths by accounting for exploration noise.

Tuple Update

Uses (S, A, R, S', A') — hence the name SARSA.

vs Q-Learning

Same convergence guarantees but with different path characteristics.

Implementation

function sarsa(env, episodes, alpha, gamma, epsilon) {
    var Q = {};
    for (var ep = 0; ep < episodes; ep++) {
        var state = env.reset();
        var action = epsilonGreedy(Q, state, epsilon);

        while (!env.done(state)) {
            var result = env.step(state, action);
            var nextState = result.nextState;
            var reward = result.reward;
            var nextAction = epsilonGreedy(Q, nextState, epsilon);

            if (!Q[state]) Q[state] = {};
            if (!Q[state][action]) Q[state][action] = 0;
            var nextQ = (Q[nextState] && Q[nextState][nextAction]) || 0;

            // On-policy update: use actual next action
            Q[state][action] += alpha * (reward + gamma * nextQ - Q[state][action]);

            state = nextState;
            action = nextAction;
        }
    }
    return Q;
}
