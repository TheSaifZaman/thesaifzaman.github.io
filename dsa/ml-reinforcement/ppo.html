Proximal Policy Optimization (PPO)

Hard Time: O(T·d²) Space: O(d²)

How It Works

Proximal Policy Optimization (PPO) is a policy gradient method that prevents destructively large policy updates by clipping the objective function. It computes the ratio r(θ) = π_new(a|s) / π_old(a|s) and clips it to stay within [1-ε, 1+ε].

The clipped objective is: L = min(r(θ) · A, clip(r(θ), 1-ε, 1+ε) · A)

This keeps updates "proximal" (close) to the previous policy, ensuring stable and monotonic improvement. PPO is the algorithm behind RLHF (Reinforcement Learning from Human Feedback) used to fine-tune large language models like ChatGPT, making it one of the most impactful RL algorithms in practice.

Clipped Objective

Prevents policy from changing too much in one update.

Stable Training

Monotonic improvement without trust region constraints.

RLHF Backbone

Core algorithm for aligning LLMs with human preferences.

Practical

Simpler than TRPO with comparable or better performance.

function ppoUpdate(trajectories, policy, valueFn, clipEpsilon, epochs) {
    // Compute advantages using GAE
    var advantages = computeGAE(trajectories, valueFn);

    // Store old policy probabilities
    var oldProbs = trajectories.map(function(t) {
        return policy.prob(t.state, t.action);
    });

    for (var epoch = 0; epoch < epochs; epoch++) {
        for (var i = 0; i < trajectories.length; i++) {
            var t = trajectories[i];
            var newProb = policy.prob(t.state, t.action);
            var ratio = newProb / oldProbs[i];

            // Clipped surrogate objective
            var unclipped = ratio * advantages[i];
            var clipped = Math.min(
                Math.max(ratio, 1 - clipEpsilon),
                1 + clipEpsilon
            ) * advantages[i];

            var policyLoss = -Math.min(unclipped, clipped);
            var valueLoss = Math.pow(valueFn(t.state) - t.returns, 2);

            // Update both networks
            policy.update(policyLoss);
            valueFn.update(valueLoss);
        }
    }
}
