Monotonic Stack & Queue

Medium Time: O(n) Space: O(n)

How It Works

A Monotonic Stack is a stack that maintains its elements in either strictly increasing or strictly decreasing order. When a new element is pushed, all elements that violate the monotonic property are popped first. This allows us to efficiently find the next greater element, next smaller element, or solve problems involving ranges bounded by extremes.

A Monotonic Queue (often implemented with a deque) extends this idea to support efficient sliding window operations. It maintains elements in monotonic order while also allowing removal from the front when elements leave the window. This enables O(1) amortized access to the window's minimum or maximum.

Both structures are powerful tools for converting O(n²) brute-force solutions into O(n) algorithms. The key insight is that each element is pushed and popped at most once, giving O(n) total time despite the inner while loop.

Key Properties

  • Amortized O(1): Each element enters and leaves the stack/queue at most once.
  • Next Greater/Smaller: Monotonic stacks directly solve "next greater element" patterns.
  • Sliding Window: Monotonic deques provide O(1) window min/max queries.
  • Histogram Problems: Essential for largest rectangle in histogram and similar area problems.

Implementation

// Next Greater Element using Monotonic Stack
function nextGreaterElements(arr) {
    var n = arr.length;
    var result = new Array(n).fill(-1);
    var stack = []; // stores indices

    for (var i = 0; i < n; i++) {
        while (stack.length > 0 && arr[stack[stack.length - 1]] < arr[i]) {
            result[stack.pop()] = arr[i];
        }
        stack.push(i);
    }
    return result;
}

// Sliding Window Maximum using Monotonic Deque
function maxSlidingWindow(nums, k) {
    var result = [];
    var deque = []; // stores indices, front has max

    for (var i = 0; i < nums.length; i++) {
        // Remove elements outside window
        while (deque.length && deque[0] <= i - k) deque.shift();
        // Maintain decreasing order
        while (deque.length && nums[deque[deque.length - 1]] <= nums[i]) deque.pop();
        deque.push(i);
        if (i >= k - 1) result.push(nums[deque[0]]);
    }
    return result;
}
