Deque (Double-Ended Queue)

Easy Push/Pop Front: O(1) Push/Pop Back: O(1) Space: O(n)

How It Works

A Deque (double-ended queue, pronounced "deck") is a generalized queue that allows insertion and removal at both the front and back in O(1) time. It combines the capabilities of both stacks (LIFO) and queues (FIFO) into a single data structure.

Deques can be implemented using a doubly linked list (each operation is O(1)) or a circular buffer / dynamic array (amortized O(1)). In JavaScript, arrays can simulate deques using push/pop for the back and unshift/shift for the front, though unshift/shift are O(n) internally. A proper implementation uses a circular buffer for true O(1) operations.

Deques are essential for the sliding window maximum/minimum pattern (monotonic deque), BFS optimizations like 0-1 BFS, and work-stealing algorithms in parallel computing. Python's collections.deque and Java's ArrayDeque are standard library implementations.

Key Properties

  • Double-Ended: O(1) insert and remove at both front and back.
  • Versatile: Can be used as a stack, queue, or both simultaneously.
  • Sliding Window: Foundation for O(n) sliding window min/max algorithms.
  • 0-1 BFS: Enables BFS on graphs with 0/1 edge weights in O(V+E).

Implementation

class Deque {
  constructor() {
    this.data = {};
    this.head = 0;
    this.tail = 0;
  }

  pushBack(val) {
    this.data[this.tail] = val;
    this.tail++;
  }

  pushFront(val) {
    this.head--;
    this.data[this.head] = val;
  }

  popBack() {
    if (this.isEmpty()) return undefined;
    this.tail--;
    var val = this.data[this.tail];
    delete this.data[this.tail];
    return val;
  }

  popFront() {
    if (this.isEmpty()) return undefined;
    var val = this.data[this.head];
    delete this.data[this.head];
    this.head++;
    return val;
  }

  peekFront() { return this.data[this.head]; }
  peekBack() { return this.data[this.tail - 1]; }
  size() { return this.tail - this.head; }
  isEmpty() { return this.tail === this.head; }
}
