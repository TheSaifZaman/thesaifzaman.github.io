Circular Buffer

Easy Enqueue: O(1) Dequeue: O(1) Space: O(n)

How It Works

A Circular Buffer (ring buffer) is a fixed-size array that wraps around: when the write pointer reaches the end, it wraps to the beginning. It uses two pointers — head (read position) and tail (write position) — and modular arithmetic to treat the array as circular. This eliminates the need to shift elements, giving O(1) enqueue and dequeue.

When the buffer is full, new writes either overwrite the oldest data (streaming mode) or reject the write (bounded mode). The full/empty distinction is handled by tracking the count or leaving one slot empty (size = capacity + 1).

Circular buffers are used in audio/video streaming, network packet buffers, producer-consumer queues, keyboard input buffers, and log rotation systems. Their fixed memory footprint and lock-free potential make them ideal for real-time and embedded systems.

Key Properties

  • Fixed Size: No dynamic memory allocation — predictable memory usage.
  • Wrap-Around: Indices use modular arithmetic: (i + 1) % capacity.
  • No Shifting: Unlike array-based queues, no elements need to be moved.
  • Lock-Free: Single-producer single-consumer variants need no locks.

Implementation

class CircularBuffer {
  constructor(capacity) {
    this.buffer = new Array(capacity);
    this.capacity = capacity;
    this.head = 0;  // read pointer
    this.tail = 0;  // write pointer
    this.count = 0;
  }

  enqueue(val) {
    if (this.isFull()) return false;
    this.buffer[this.tail] = val;
    this.tail = (this.tail + 1) % this.capacity;
    this.count++;
    return true;
  }

  dequeue() {
    if (this.isEmpty()) return undefined;
    var val = this.buffer[this.head];
    this.buffer[this.head] = undefined;
    this.head = (this.head + 1) % this.capacity;
    this.count--;
    return val;
  }

  peek() { return this.isEmpty() ? undefined : this.buffer[this.head]; }
  isFull() { return this.count === this.capacity; }
  isEmpty() { return this.count === 0; }
  size() { return this.count; }
}
