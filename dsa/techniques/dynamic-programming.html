Dynamic Programming

Hard Time: Varies Space: Varies

How It Works

Dynamic Programming (DP) solves complex problems by breaking them into overlapping sub-problems and storing their results to avoid redundant computation. Unlike divide and conquer where sub-problems are independent, DP sub-problems share common sub-sub-problems, making memoization or tabulation essential.

The two approaches are top-down (memoization), where you solve recursively and cache results, and bottom-up (tabulation), where you fill a table iteratively from the smallest sub-problems upward. Bottom-up avoids recursion overhead and is often preferred in interviews.

Identifying a DP problem requires recognizing optimal substructure (optimal solution uses optimal sub-solutions) and overlapping sub-problems (same sub-problem solved multiple times). The hardest part is defining the state: what information do you need to make a decision at each step?

Optimal Substructure
The optimal solution can be constructed from optimal solutions of its sub-problems.
Overlapping Sub-problems
The same sub-problems are solved repeatedly — caching prevents exponential blowup.
State Definition
Define dp[i] (or dp[i][j]) clearly: what does each cell represent? This is the hardest step.
Transition Formula
Express dp[i] in terms of previously computed values. This is the recurrence relation.

// Climbing Stairs (LeetCode #70)
function climbStairs(n) {
    if (n <= 2) return n;
    var prev2 = 1, prev1 = 2;
    for (var i = 3; i <= n; i++) {
        var curr = prev1 + prev2;
        prev2 = prev1;
        prev1 = curr;
    }
    return prev1;
}

// Coin Change (LeetCode #322)
function coinChange(coins, amount) {
    var dp = new Array(amount + 1).fill(Infinity);
    dp[0] = 0;
    for (var i = 1; i <= amount; i++) {
        for (var j = 0; j < coins.length; j++) {
            if (coins[j] <= i && dp[i - coins[j]] + 1 < dp[i]) {
                dp[i] = dp[i - coins[j]] + 1;
            }
        }
    }
    return dp[amount] === Infinity ? -1 : dp[amount];
}

// Longest Common Subsequence (LeetCode #1143)
function longestCommonSubsequence(text1, text2) {
    var m = text1.length, n = text2.length;
    var dp = [];
    for (var i = 0; i <= m; i++) dp[i] = new Array(n + 1).fill(0);
    for (var i = 1; i <= m; i++) {
        for (var j = 1; j <= n; j++) {
            if (text1[i-1] === text2[j-1]) dp[i][j] = dp[i-1][j-1] + 1;
            else dp[i][j] = Math.max(dp[i-1][j], dp[i][j-1]);
        }
    }
    return dp[m][n];
}

// House Robber (LeetCode #198)
function rob(nums) {
    if (nums.length === 0) return 0;
    var prev2 = 0, prev1 = 0;
    for (var i = 0; i < nums.length; i++) {
        var curr = Math.max(prev1, prev2 + nums[i]);
        prev2 = prev1;
        prev1 = curr;
    }
    return prev1;
}
