Recursion

Easy Time: Varies Space: O(n) stack

How It Works

Recursion is a technique where a function calls itself to solve smaller instances of the same problem. Every recursive solution has a base case (the simplest instance solved directly) and a recursive case (breaking the problem into smaller instances).

The call stack maintains the state of each recursive call. When the base case is reached, the stack unwinds, combining results from the deepest level upward. Understanding the call stack is crucial for debugging and analyzing space complexity.

Common recursive patterns include: linear recursion (factorial, linked list traversal), binary recursion (tree traversals, Fibonacci), and multiple recursion (permutations, combinations). Tail recursion, where the recursive call is the last operation, can be optimized by compilers into iteration.

Base Case
The stopping condition that prevents infinite recursion. Every recursive function needs at least one.
Recursive Case
Breaks the problem into a smaller version and calls itself. Must progress toward the base case.
Call Stack
Each recursive call adds a frame to the stack, using O(n) space for n-deep recursion.
Tail Recursion
When the recursive call is the last operation, the compiler can optimize to O(1) stack space.

Implementation

// Factorial — classic linear recursion
function factorial(n) {
    if (n <= 1) return 1;  // base case
    return n * factorial(n - 1);  // recursive case
}

// Fibonacci — binary recursion
function fibonacci(n) {
    if (n <= 1) return n;
    return fibonacci(n - 1) + fibonacci(n - 2);
}

// Power function — divide and conquer recursion
function power(base, exp) {
    if (exp === 0) return 1;
    if (exp % 2 === 0) {
        var half = power(base, exp / 2);
        return half * half;
    }
    return base * power(base, exp - 1);
}

// Reverse a string recursively
function reverseString(s) {
    if (s.length <= 1) return s;
    return reverseString(s.substring(1)) + s[0];
}
