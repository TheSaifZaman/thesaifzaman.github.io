Meet in the Middle

Hard Time: O(2^(n/2)) Space: O(2^(n/2))

How It Works

Meet in the Middle splits the input into two halves, exhaustively explores all possibilities for each half independently, then combines results. This reduces exponential complexity from O(2^n) to O(2^(n/2)), which is a massive improvement for n around 30-40.

For example, to find a subset with a target sum from n=36 elements: brute force is O(2^36) which is impractical. But splitting into two halves of 18 elements each: enumerate all 2^18 subset sums for each half (~260K each), sort one half, then binary search for complements. Total: O(2^18 * 18).

The technique applies to: subset sum with large n, closest subset sum, counting pairs with specific properties, and certain cryptographic attacks. The core pattern is always: split, enumerate both halves, combine with sorting + binary search.

Split in Half
Divide the input into two roughly equal halves (n/2 elements each).
Enumerate Both
Generate all 2^(n/2) possibilities for each half independently.
Combine with Search
Sort one half's results, binary search for complements from the other half.
O(2^n) to O(2^(n/2))
Reduces 2^40 (~10^12) to 2^20 (~10^6), making previously impossible problems tractable.

Implementation

// Meet in the Middle — subset sum closest to target
function closestSubsetSum(arr, target) {
    var n = arr.length;
    var half = Math.floor(n / 2);

    // Generate all subset sums for each half
    function allSubsetSums(a) {
        var sums = [0];
        for (var i = 0; i < a.length; i++) {
            var len = sums.length;
            for (var j = 0; j < len; j++) {
                sums.push(sums[j] + a[i]);
            }
        }
        return sums;
    }

    var leftSums = allSubsetSums(arr.slice(0, half));
    var rightSums = allSubsetSums(arr.slice(half));

    rightSums.sort(function(a, b) { return a - b; });

    var best = Infinity;
    for (var i = 0; i < leftSums.length; i++) {
        var need = target - leftSums[i];
        // Binary search for closest in rightSums
        var lo = 0, hi = rightSums.length - 1;
        while (lo <= hi) {
            var mid = (lo + hi) >> 1;
            var diff = Math.abs(leftSums[i] + rightSums[mid] - target);
            best = Math.min(best, diff);
            if (rightSums[mid] < need) lo = mid + 1;
            else hi = mid - 1;
        }
        if (lo < rightSums.length) {
            best = Math.min(best, Math.abs(leftSums[i] + rightSums[lo] - target));
        }
    }
    return target - best; // or target + best depending on direction
}
