Ternary Search

Divides search space into thirds to find target or unimodal extrema — O(log n)

Concept

Ternary Search divides the search range into three equal parts using two midpoints (mid1 and mid2). It compares the target with values at both midpoints to eliminate one-third of the search space each step.

  • Time Complexity: O(log3 n) = O(log n)
  • Space Complexity: O(1) iterative
  • Precondition: Array must be sorted (or function must be unimodal)
  • Key Idea: Two midpoints divide range into thirds; eliminate one third per step

Implementation

function ternarySearch(arr, target) {
  let left = 0;
  let right = arr.length - 1;

  while (left <= right) {
    const third = Math.floor((right - left) / 3);
    const mid1 = left + third;
    const mid2 = right - third;

    if (arr[mid1] === target) return mid1;
    if (arr[mid2] === target) return mid2;

    if (target < arr[mid1]) {
      right = mid1 - 1;       // Search first third
    } else if (target > arr[mid2]) {
      left = mid2 + 1;        // Search last third
    } else {
      left = mid1 + 1;        // Search middle third
      right = mid2 - 1;
    }
  }

  return -1;                   // Not found
}
