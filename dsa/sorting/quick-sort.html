Quick Sort

Medium Best: O(n log n) Avg: O(n log n) Worst: O(n²) Space: O(log n)

How It Works

Quick Sort is a divide-and-conquer algorithm that selects a "pivot" element and partitions the array into two sub-arrays: elements less than the pivot and elements greater than the pivot. It then recursively sorts the sub-arrays.

The partition step is the heart of Quick Sort. Using the Lomuto or Hoare partition scheme, elements are rearranged so that all elements smaller than the pivot come before it and all larger elements come after it. The pivot is then in its final sorted position.

Quick Sort is often the fastest general-purpose sorting algorithm in practice due to excellent cache locality and low constant factors. Its O(n²) worst case occurs with poor pivot choices (e.g., already sorted input with first-element pivot), but this can be mitigated with randomized or median-of-three pivot selection.

Key Properties

  • Stable: No — partitioning can change relative order of equal elements.
  • In-place: Yes — O(log n) stack space for recursion.
  • Adaptive: No — but randomized pivot avoids worst case in practice.
  • Cache-friendly: Yes — sequential access patterns.

Implementation

function quickSort(arr, lo, hi) {
    if (lo === undefined) lo = 0;
    if (hi === undefined) hi = arr.length - 1;
    if (lo < hi) {
        var p = partition(arr, lo, hi);
        quickSort(arr, lo, p - 1);
        quickSort(arr, p + 1, hi);
    }
    return arr;
}

function partition(arr, lo, hi) {
    var pivot = arr[hi];
    var i = lo - 1;
    for (var j = lo; j < hi; j++) {
        if (arr[j] <= pivot) {
            i++;
            var temp = arr[i];
            arr[i] = arr[j];
            arr[j] = temp;
        }
    }
    var temp2 = arr[i + 1];
    arr[i + 1] = arr[hi];
    arr[hi] = temp2;
    return i + 1;
}
