Merge Sort

Medium Best: O(n log n) Avg: O(n log n) Worst: O(n log n) Space: O(n)

How It Works

Merge Sort is a divide-and-conquer algorithm that splits the array into halves, recursively sorts each half, and then merges the two sorted halves back together. The merge operation is the key step: it combines two sorted arrays into a single sorted array in linear time.

The algorithm guarantees O(n log n) performance in all cases (best, average, and worst), making it more predictable than Quick Sort. The divide step takes O(log n) levels of recursion, and at each level, the merge step processes all n elements.

Merge Sort is particularly useful for sorting linked lists (where it achieves O(1) extra space) and for external sorting of large datasets that do not fit in memory. Its main drawback for arrays is the O(n) extra space required for the merge step.

Key Properties

  • Stable: Yes — equal elements maintain their relative order.
  • In-place: No — requires O(n) auxiliary space for arrays.
  • Adaptive: No — always O(n log n) regardless of input.
  • Parallelizable: Yes — sub-problems are independent.

Implementation

function mergeSort(arr) {
    if (arr.length <= 1) return arr;
    var mid = Math.floor(arr.length / 2);
    var left = mergeSort(arr.slice(0, mid));
    var right = mergeSort(arr.slice(mid));
    return merge(left, right);
}

function merge(left, right) {
    var result = [], i = 0, j = 0;
    while (i < left.length && j < right.length) {
        if (left[i] <= right[j]) {
            result.push(left[i++]);
        } else {
            result.push(right[j++]);
        }
    }
    return result.concat(left.slice(i)).concat(right.slice(j));
}
