Shell Sort

Medium Best: O(n log n) Avg: O(n^1.5) Worst: O(n^2) Space: O(1)

How It Works

Shell Sort is a generalization of Insertion Sort that allows the exchange of elements that are far apart. It works by sorting elements at specific intervals (gaps) and progressively reducing the gap until it becomes 1, at which point it performs a standard Insertion Sort on a nearly-sorted array.

The key insight is that Insertion Sort is very efficient on nearly-sorted data. By first sorting elements that are far apart, Shell Sort moves elements closer to their final positions quickly. The gap sequence determines the algorithm's performance; common sequences include n/2 (Shell's original), Knuth's (3^k-1)/2, and Sedgewick's sequence.

Shell Sort's time complexity depends on the gap sequence used. With Shell's original sequence (n/2, n/4, ..., 1), the worst case is O(n²), but better sequences achieve O(n^(4/3)) or even O(n log² n). It is an in-place algorithm with good cache performance.

Key Properties

  • Stable: No — elements can jump over equal elements during gap-based sorting.
  • In-place: Yes — O(1) extra space.
  • Adaptive: Yes — benefits from partially sorted input.
  • Gap-dependent: Performance varies significantly with gap sequence choice.

Implementation

function shellSort(arr) {
    var n = arr.length;
    // Using Shell's original gap sequence: n/2, n/4, ..., 1
    for (var gap = Math.floor(n / 2); gap > 0; gap = Math.floor(gap / 2)) {
        for (var i = gap; i < n; i++) {
            var temp = arr[i];
            var j = i;
            while (j >= gap && arr[j - gap] > temp) {
                arr[j] = arr[j - gap];
                j -= gap;
            }
            arr[j] = temp;
        }
    }
    return arr;
}
