Heap Sort

Medium Best: O(n log n) Avg: O(n log n) Worst: O(n log n) Space: O(1)

How It Works

Heap Sort works in two phases. First, it builds a max-heap from the input array. A max-heap is a complete binary tree where every parent node is greater than or equal to its children. This can be done in O(n) time using the "heapify" procedure from the bottom up.

In the second phase, it repeatedly extracts the maximum element (root of the heap) by swapping it with the last element of the heap, reducing the heap size by one, and then restoring the heap property by "sifting down" the new root. Each extraction takes O(log n) time.

Heap Sort guarantees O(n log n) worst-case performance with O(1) extra space, making it useful when guaranteed performance and minimal memory are required. However, it is not stable and has poor cache locality compared to Quick Sort, making it slower in practice despite the same asymptotic complexity.

Key Properties

  • Stable: No — heap operations can change relative order of equal elements.
  • In-place: Yes — O(1) extra space.
  • Adaptive: No — always O(n log n).
  • Guaranteed performance: O(n log n) worst case, unlike Quick Sort.

Implementation

function heapSort(arr) {
    var n = arr.length;

    // Build max-heap
    for (var i = Math.floor(n / 2) - 1; i >= 0; i--) {
        heapify(arr, n, i);
    }

    // Extract elements one by one
    for (var i = n - 1; i > 0; i--) {
        var temp = arr[0];
        arr[0] = arr[i];
        arr[i] = temp;
        heapify(arr, i, 0);
    }
    return arr;
}

function heapify(arr, size, root) {
    var largest = root;
    var left = 2 * root + 1;
    var right = 2 * root + 2;
    if (left < size && arr[left] > arr[largest]) largest = left;
    if (right < size && arr[right] > arr[largest]) largest = right;
    if (largest !== root) {
        var temp = arr[root];
        arr[root] = arr[largest];
        arr[largest] = temp;
        heapify(arr, size, largest);
    }
}
