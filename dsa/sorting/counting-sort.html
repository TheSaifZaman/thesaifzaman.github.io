Counting Sort

Easy Best: O(n+k) Avg: O(n+k) Worst: O(n+k) Space: O(n+k)

How It Works

Counting Sort is a non-comparison-based sorting algorithm that works by counting the number of occurrences of each distinct element. It then uses these counts to determine the position of each element in the sorted output. The "k" in O(n+k) represents the range of input values.

The algorithm first finds the range of input values and creates a count array of that size. It then counts occurrences of each value, computes prefix sums to determine positions, and finally places each element into its correct sorted position by iterating backward through the input (to maintain stability).

Counting Sort is extremely efficient when the range of input values (k) is not significantly greater than the number of elements (n). It is commonly used as a subroutine in Radix Sort. Its main limitation is that it only works with non-negative integers (or values that can be mapped to non-negative integers).

Key Properties

  • Stable: Yes — preserves relative order of equal elements.
  • In-place: No — requires O(n+k) extra space.
  • Non-comparison: Does not compare elements directly.
  • Linear time: O(n+k) when k = O(n).

Implementation

function countingSort(arr) {
    var max = Math.max.apply(null, arr);
    var min = Math.min.apply(null, arr);
    var range = max - min + 1;
    var count = new Array(range).fill(0);
    var output = new Array(arr.length);

    // Count occurrences
    for (var i = 0; i < arr.length; i++) {
        count[arr[i] - min]++;
    }

    // Prefix sums
    for (var i = 1; i < range; i++) {
        count[i] += count[i - 1];
    }

    // Build output (iterate backward for stability)
    for (var i = arr.length - 1; i >= 0; i--) {
        output[count[arr[i] - min] - 1] = arr[i];
        count[arr[i] - min]--;
    }

    return output;
}
