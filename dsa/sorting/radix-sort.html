Radix Sort

Medium Best: O(d·(n+k)) Avg: O(d·(n+k)) Worst: O(d·(n+k)) Space: O(n+k)

How It Works

Radix Sort processes integer keys digit by digit, from the least significant digit (LSD) to the most significant digit (MSD). At each digit position, it uses a stable sort (typically Counting Sort) as a subroutine to sort elements based on that digit alone.

The key insight is that if we sort by each digit position using a stable sort, the overall order is maintained correctly. After processing all d digits (where d is the number of digits in the largest number), the entire array is sorted. The variable k represents the base (typically 10 for decimal).

Radix Sort achieves linear time O(d(n+k)) when the number of digits d is constant. It is particularly efficient for sorting integers, strings of fixed length, or any data that can be decomposed into fixed-size keys. It avoids the O(n log n) lower bound of comparison-based sorts.

Key Properties

  • Stable: Yes — relies on stable subroutine sort.
  • In-place: No — requires O(n+k) auxiliary space.
  • Non-comparison: Processes digits, not element comparisons.
  • Linear time: When d and k are constants relative to n.

Implementation

function radixSort(arr) {
    var max = Math.max.apply(null, arr);

    for (var exp = 1; Math.floor(max / exp) > 0; exp *= 10) {
        countingSortByDigit(arr, exp);
    }
    return arr;
}

function countingSortByDigit(arr, exp) {
    var n = arr.length;
    var output = new Array(n);
    var count = new Array(10).fill(0);

    for (var i = 0; i < n; i++) {
        var digit = Math.floor(arr[i] / exp) % 10;
        count[digit]++;
    }

    for (var i = 1; i < 10; i++) {
        count[i] += count[i - 1];
    }

    for (var i = n - 1; i >= 0; i--) {
        var digit = Math.floor(arr[i] / exp) % 10;
        output[count[digit] - 1] = arr[i];
        count[digit]--;
    }

    for (var i = 0; i < n; i++) {
        arr[i] = output[i];
    }
}
