How It Works

Complexity analysis is the mathematical framework for measuring how an algorithm's resource consumption (time or memory) grows as the input size increases. Rather than timing code on a specific machine, we express performance in terms of the input size n, giving us a machine-independent way to compare algorithms.

The three main asymptotic notations are: Big O (O) — the upper bound (worst case), Big Omega (Ω) — the lower bound (best case), and Big Theta (Θ) — the tight bound (average case when upper and lower match). In practice, Big O is used most frequently because we want to understand the worst-case scenario of our algorithms.

Common complexity classes, from fastest to slowest, are: O(1) constant, O(log n) logarithmic, O(n) linear, O(n log n) linearithmic, O(n²) quadratic, and O(2n) exponential. Understanding where your algorithm falls in this hierarchy is crucial for writing code that scales. A quadratic algorithm might work fine for n=100 but become unusable at n=100,000.

Big O Notation
Describes the upper bound of growth. Drop constants and lower-order terms: O(3n + 5) simplifies to O(n).
Analyzing Loops
A single loop over n items is O(n). Nested loops multiply: a loop inside a loop is O(n²). A loop that halves n each iteration is O(log n).
Space Complexity
Measures additional memory used. An algorithm using a fixed number of variables is O(1) space; one that creates an array of size n is O(n) space.
Recursion Analysis
Use the Master Theorem or recursion trees. Each recursive call adds to the call stack, contributing to both time and space complexity.

Implementation

// O(1) — Constant time
function getFirst(arr) {
    return arr[0];
}

// O(log n) — Logarithmic (binary search)
function binarySearch(arr, target) {
    var lo = 0, hi = arr.length - 1;
    while (lo <= hi) {
        var mid = Math.floor((lo + hi) / 2);
        if (arr[mid] === target) return mid;
        if (arr[mid] < target) lo = mid + 1;
        else hi = mid - 1;
    }
    return -1;
}

// O(n) — Linear
function findMax(arr) {
    var max = arr[0];
    for (var i = 1; i < arr.length; i++) {
        if (arr[i] > max) max = arr[i];
    }
    return max;
}

// O(n log n) — Linearithmic (merge sort)
function mergeSort(arr) {
    if (arr.length <= 1) return arr;
    var mid = Math.floor(arr.length / 2);
    var left = mergeSort(arr.slice(0, mid));
    var right = mergeSort(arr.slice(mid));
    return merge(left, right);
}

// O(n^2) — Quadratic (bubble sort)
function bubbleSort(arr) {
    for (var i = 0; i < arr.length; i++) {
        for (var j = 0; j < arr.length - i - 1; j++) {
            if (arr[j] > arr[j + 1]) {
                var tmp = arr[j];
                arr[j] = arr[j + 1];
                arr[j + 1] = tmp;
            }
        }
    }
    return arr;
}

// O(2^n) — Exponential (naive Fibonacci)
function fibonacci(n) {
    if (n <= 1) return n;
    return fibonacci(n - 1) + fibonacci(n - 2);
}
