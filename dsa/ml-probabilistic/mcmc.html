Markov Chain Monte Carlo

Hard Time: O(n·T) Space: O(T)

How It Works

Markov Chain Monte Carlo (MCMC) generates samples from a target probability distribution by constructing a Markov chain whose stationary distribution equals the target. The Metropolis-Hastings algorithm proposes a move from the current state, then accepts or rejects it based on the acceptance ratio.

After a burn-in period, the chain samples approximate the target distribution. MCMC is essential for Bayesian inference where direct sampling is intractable, enabling computation of posterior expectations, credible intervals, and model comparison.

Key Properties

  • Proposal: Random walk, Gaussian, or problem-specific proposal distributions.
  • Acceptance ratio: α = min(1, P(x')/P(x) · q(x|x')/q(x'|x)).
  • Burn-in: Initial samples discarded before the chain converges.
  • Diagnostics: Trace plots, autocorrelation, R-hat for convergence checks.

Implementation

function metropolisHastings(targetPDF, proposal, x0, nSamples) {
    var samples = [x0];
    var current = x0;
    for (var i = 0; i < nSamples; i++) {
        var candidate = proposal(current);
        var alpha = Math.min(1, targetPDF(candidate) / targetPDF(current));
        if (Math.random() < alpha) {
            current = candidate; // Accept
        }
        // else: stay at current (reject)
        samples.push(current);
    }
    return samples;
}
