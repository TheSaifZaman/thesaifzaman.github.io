Hidden Markov Models

Hard Viterbi: O(T·S²) Space: O(T·S)

How It Works

A Hidden Markov Model (HMM) models a system with hidden states that produce observable outputs. At each time step, the system transitions from one hidden state to another according to transition probabilities, and emits an observation from the new state according to emission probabilities.

The three fundamental problems are: (1) Evaluation — compute P(observations | model) using the Forward algorithm; (2) Decoding — find the most likely hidden state sequence using the Viterbi algorithm; (3) Learning — estimate model parameters using Baum-Welch (EM).

Key Properties

  • Markov property: Next state depends only on current state, not history.
  • Hidden states: Not directly observable; inferred from emissions.
  • Parameters: Initial π, transition A, emission B matrices.
  • Applications: Speech recognition, POS tagging, gene finding, gesture recognition.

function viterbi(obs, states, startP, transP, emitP) {
    var T = obs.length, N = states.length;
    var dp = [], path = [];
    // Initialize
    dp[0] = [];
    path[0] = [];
    for (var s = 0; s < N; s++) {
        dp[0][s] = startP[s] * emitP[s][obs[0]];
        path[0][s] = s;
    }
    // Recurse
    for (var t = 1; t < T; t++) {
        dp[t] = []; path[t] = [];
        for (var j = 0; j < N; j++) {
            var maxP = -1, bestS = 0;
            for (var i = 0; i < N; i++) {
                var p = dp[t-1][i] * transP[i][j] * emitP[j][obs[t]];
                if (p > maxP) { maxP = p; bestS = i; }
            }
            dp[t][j] = maxP;
            path[t][j] = bestS;
        }
    }
    // Backtrack
    return backtrack(dp, path, T, N);
}
