Stochastic Gradient Descent

Easy Time: O(d) per update Space: O(d)

How It Works

Stochastic Gradient Descent (SGD) updates parameters using the gradient from a single random sample (or small mini-batch) rather than the entire dataset. This makes each update much faster but noisier.

The noisy path has a surprising benefit: it can help escape local minima and saddle points that trap batch gradient descent. The noise acts as implicit regularization, often leading to better generalization.

Mini-batch SGD (the practical standard) uses batches of 32-256 samples, balancing computation efficiency with gradient quality. Most deep learning training uses some form of SGD.

Fast Updates

Each step uses one sample — much faster per iteration.

Noisy Gradients

Random sampling adds noise that helps escape local minima.

Mini-Batch

Practical compromise: batch of 32-256 for stable GPU utilization.

Implicit Regularization

Noise prevents overfitting, improving generalization.

Implementation

function sgd(X, y, learningRate, epochs) {
    var theta = new Array(X[0].length).fill(0);
    var m = X.length;

    for (var epoch = 0; epoch < epochs; epoch++) {
        // Shuffle data each epoch
        var indices = shuffle(range(m));

        for (var i = 0; i < m; i++) {
            var idx = indices[i];
            var prediction = dotProduct(X[idx], theta);
            var error = prediction - y[idx];

            // Update using single sample gradient
            for (var j = 0; j < theta.length; j++) {
                theta[j] -= learningRate * error * X[idx][j];
            }
        }
    }
    return theta;
}
