Gradient Descent

Easy Time: O(n·d·iterations) Space: O(d)

How It Works

Gradient Descent is the foundational optimization algorithm in machine learning. It iteratively adjusts parameters by moving in the direction of steepest decrease of the loss function. The update rule is:

θ ← θ - α · ∇L(θ)

where α is the learning rate and ∇L is the gradient of the loss function. The gradient points in the direction of steepest ascent, so we subtract it to descend toward the minimum. Think of it as a ball rolling downhill on a loss surface — it naturally moves toward the lowest point.

In Batch Gradient Descent, the gradient is computed over the entire dataset at each step, giving a smooth but potentially slow path. The learning rate α controls step size: too large and it overshoots, too small and convergence is slow.

Steepest Descent

Always moves in the direction that reduces loss fastest locally.

Learning Rate

Controls step size — critical hyperparameter for convergence.

Convex Guarantee

Converges to global minimum for convex loss functions.

Local Minima

May get stuck in local minima for non-convex functions.

Implementation

function gradientDescent(X, y, learningRate, iterations) {
    var m = X.length;
    var theta = new Array(X[0].length).fill(0);

    for (var iter = 0; iter < iterations; iter++) {
        // Compute gradient over entire dataset
        var gradient = new Array(theta.length).fill(0);
        for (var i = 0; i < m; i++) {
            var prediction = dotProduct(X[i], theta);
            var error = prediction - y[i];
            for (var j = 0; j < theta.length; j++) {
                gradient[j] += (error * X[i][j]) / m;
            }
        }

        // Update parameters
        for (var j = 0; j < theta.length; j++) {
            theta[j] -= learningRate * gradient[j];
        }
    }
    return theta;
}
