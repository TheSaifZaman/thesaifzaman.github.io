Adam Optimizer

Medium Time: O(d) per update Space: O(d)

How It Works

Adam (Adaptive Moment Estimation) combines the benefits of Momentum (first moment) and RMSProp (second moment) into a single optimizer. It maintains per-parameter adaptive learning rates, making it robust across a wide range of problems.

Adam tracks two exponential moving averages: m (first moment, like momentum) and v (second moment, like RMSProp). With bias correction applied, the update is: θ -= α · m̂ / (√v̂ + ε)

Parameters with large gradients get smaller effective learning rates (via v normalization), and parameters with consistent gradient direction get acceleration (via m). This makes Adam the default optimizer for most deep learning tasks.

Adaptive LR

Each parameter gets its own learning rate based on gradient history.

Momentum

First moment (m) accumulates gradient direction like momentum.

RMSProp

Second moment (v) normalizes by gradient magnitude.

Bias Correction

Corrects initial bias when m and v are initialized to zero.

Implementation

function adam(gradFn, theta, lr, beta1, beta2, epsilon, iterations) {
    var m = new Array(theta.length).fill(0); // first moment
    var v = new Array(theta.length).fill(0); // second moment
    beta1 = beta1 || 0.9;
    beta2 = beta2 || 0.999;
    epsilon = epsilon || 1e-8;

    for (var t = 1; t <= iterations; t++) {
        var grad = gradFn(theta);

        for (var i = 0; i < theta.length; i++) {
            // Update moments
            m[i] = beta1 * m[i] + (1 - beta1) * grad[i];
            v[i] = beta2 * v[i] + (1 - beta2) * grad[i] * grad[i];

            // Bias correction
            var mHat = m[i] / (1 - Math.pow(beta1, t));
            var vHat = v[i] / (1 - Math.pow(beta2, t));

            // Update parameter
            theta[i] -= lr * mHat / (Math.sqrt(vHat) + epsilon);
        }
    }
    return theta;
}
