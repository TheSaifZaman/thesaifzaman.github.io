Gated Recurrent Unit (GRU)

Hard Time: O(T·h²) Params: 3× vanilla RNN

How It Works

GRU (Gated Recurrent Unit) is a simplified version of LSTM that combines the forget and input gates into a single update gate and merges the cell state and hidden state. It achieves comparable performance to LSTM with fewer parameters.

GRU has two gates:

  • Reset Gate (r): Controls how much of the previous hidden state to forget when computing the candidate. r = sigmoid(W_r * [h_{t-1}, x_t])
  • Update Gate (z): Controls the balance between keeping the old hidden state and adopting the new candidate. z = sigmoid(W_z * [h_{t-1}, x_t])

The update: h_t = (1 - z) * h_{t-1} + z * tanh(W * [r * h_{t-1}, x_t]). When z is close to 1, the GRU mostly uses the new candidate. When z is close to 0, it mostly keeps the old state.

Key Properties

  • Simpler than LSTM: 2 gates instead of 3, no separate cell state.
  • Fewer Parameters: 3x vanilla RNN parameters vs LSTM's 4x.
  • Comparable Performance: Often matches LSTM on many tasks.
  • Faster Training: Less computation per step than LSTM.

Implementation

function GRUCell(inputSize, hiddenSize) {
    var Wz = randomMatrix(hiddenSize, inputSize + hiddenSize);
    var Wr = randomMatrix(hiddenSize, inputSize + hiddenSize);
    var Wh = randomMatrix(hiddenSize, inputSize + hiddenSize);

    this.forward = function(x, hPrev) {
        var concat = hPrev.concat(x);
        var z = sigmoid(matVec(Wz, concat));   // update gate
        var r = sigmoid(matVec(Wr, concat));   // reset gate
        var rh = multiply(r, hPrev).concat(x);
        var hCand = tanh(matVec(Wh, rh));      // candidate

        // h_t = (1-z)*h_{t-1} + z*hCand
        var h = add(multiply(onesMinus(z), hPrev), multiply(z, hCand));
        return h;
    };
}
