Decision Tree

Easy Train: O(n log n) Predict: O(log n) Space: O(nodes)

How It Works

A Decision Tree recursively splits the feature space into regions using if-then rules. At each internal node, it selects the feature and threshold that best separates the classes (using Gini impurity or information gain). Leaf nodes contain the final class predictions.

The tree grows by greedily choosing the best split at each step until a stopping criterion is met (max depth, min samples, or pure leaves).

Math Formula

  • Gini Impurity: G = 1 − Σk pk²
  • Entropy: H = −Σk pk log2(pk)
  • Information Gain: IG = H(parent) − Σ (nchild/n) H(child)
  • Split criterion: Choose feature & threshold maximizing IG (or minimizing Gini).

Key Properties

  • Interpretable: Easy to visualize and explain decisions.
  • Non-parametric: No assumptions about data distribution.
  • Handles mixed types: Works with both numerical and categorical features.
  • Prone to overfitting: Deep trees memorize training data. Prune or limit depth.

function giniImpurity(labels) {
    var counts = {}, n = labels.length;
    for (var i = 0; i < n; i++) {
        counts[labels[i]] = (counts[labels[i]] || 0) + 1;
    }
    var gini = 1;
    for (var k in counts) {
        var p = counts[k] / n;
        gini -= p * p;
    }
    return gini;
}
