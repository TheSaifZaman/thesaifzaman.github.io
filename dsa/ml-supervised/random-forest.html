Random Forest

Medium Train: O(k·n log n) Predict: O(k·log n) Space: O(k·nodes)

How It Works

Random Forest is an ensemble of decision trees trained on random subsets of the data (bagging) and random subsets of features. Each tree votes on the class, and the majority vote determines the final prediction.

By combining many weak learners with different biases, Random Forest reduces variance without significantly increasing bias. The randomness in data and feature selection decorrelates the trees, making the ensemble more robust.

Math Formula

  • Bagging: Train each tree on a bootstrap sample of size n.
  • Feature subsample: At each split, consider only √d random features.
  • Prediction: ŷ = mode(h1(x), h2(x), ..., hk(x))
  • OOB Error: Each tree is tested on ~37% of samples it didn't train on.

Key Properties

  • Low variance: Averaging reduces overfitting from individual trees.
  • Feature importance: Measures how much each feature contributes.
  • Parallelizable: Trees are independent; can train in parallel.
  • Robust: Works well out-of-the-box with minimal tuning.

Implementation

function randomForest(data, labels, nTrees, maxDepth) {
    var trees = [];
    for (var t = 0; t < nTrees; t++) {
        var sample = bootstrapSample(data, labels);
        var tree = buildTree(sample.data, sample.labels, 0, maxDepth, true);
        trees.push(tree);
    }
    return {
        predict: function(x) {
            var votes = {};
            for (var i = 0; i < trees.length; i++) {
                var pred = treePredict(trees[i], x);
                votes[pred] = (votes[pred] || 0) + 1;
            }
            return majorityVote(votes);
        }
    };
}
