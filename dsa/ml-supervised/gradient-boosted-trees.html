Gradient Boosted Trees

Hard Train: O(k·n log n) Predict: O(k·log n) Space: O(k·nodes)

How It Works

Gradient Boosted Trees build an ensemble of decision trees sequentially. Each new tree is trained on the residual errors (negative gradient of the loss function) of the previous ensemble. The final prediction is the sum of all trees' outputs, weighted by a learning rate.

Unlike Random Forest (parallel bagging), boosting is sequential — each tree explicitly corrects the mistakes of its predecessors. This additive approach produces a strong learner from many weak learners.

Math Formula

  • Ensemble: Fm(x) = Fm−1(x) + η · hm(x)
  • Residuals: rim = −∂L(yi, F(xi)) / ∂F(xi)
  • Learning rate: η (0 < η ≤ 1) shrinks each tree's contribution.
  • Loss functions: MSE (regression), log-loss (classification), etc.

Key Properties

  • State-of-the-art: Top performer in Kaggle competitions (XGBoost, LightGBM).
  • Sequential training: Each tree corrects previous errors.
  • Regularization: Learning rate, max depth, subsampling prevent overfitting.
  • Feature importance: Provides reliable feature rankings.

Implementation

function gradientBoosting(X, y, nTrees, lr, maxDepth) {
    var predictions = new Array(y.length).fill(mean(y));
    var trees = [];
    for (var m = 0; m < nTrees; m++) {
        // Compute residuals
        var residuals = y.map(function(yi, i) {
            return yi - predictions[i];
        });
        // Fit tree to residuals
        var tree = buildTree(X, residuals, 0, maxDepth);
        trees.push(tree);
        // Update predictions
        for (var i = 0; i < X.length; i++) {
            predictions[i] += lr * treePredict(tree, X[i]);
        }
    }
    return trees;
}
